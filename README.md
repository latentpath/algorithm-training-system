# 算法训练体系使用说明

# 一、这是什么？

这是一套系统化算法训练体系。目标是建立真实面试中的：

- 算法模型识别能力
- 稳定实现能力
- 条件反射级判断能力
- 结构迁移能力

# 二、这套体系适合谁？

- 学过算法，但缺乏系统训练的人
- 已经刷过题，但能力不稳定的人
- 准备大厂面试的人

# 三、这套体系的核心理念

算法训练的核心是做到：

看到题目 → 快速判断模型 → 稳定写出 → 解释清楚

本体系重点训练以下能力：

- 模型识别：快速判断算法
- 模板稳定性：稳定写出代码
- 边界控制：减少低级错误
- 结构迁移：举一反三
- 复盘能力：建立长期记忆

# 四、整个体系分为两个阶段

## 第一阶段：算法专题训练

目标：建立高频算法模型肌肉记忆。

核心内容包括数组、链表、哈希、双指针、滑窗、BFS/DFS、树、DP、回溯、图、单调结构、Heap、贪心等高频专题。

第一阶段重点是把高频模型写稳。

训练目标：知道什么时候用什么模型。

## 第二阶段：算法结构认知

目标：从“题型”升级到“结构”。

核心内容包括信息流、状态结构、边界结构、搜索结构、递归返回、状态转移、单调维护、结构组合等。

第二阶段重点是理解算法背后的统一结构。

训练目标：不同题型之间建立迁移能力（例如把滑窗、前缀和、双指针统一理解为连续区间结构）。

# 五、推荐学习顺序

正确顺序是：

第一阶段 → 每日训练 → 第二阶段

不要跳过第一阶段直接进入第二阶段。第二阶段需要你已经具备基础模板能力、常见题型经验、Debug经验和基础复杂度意识。

# 六、如何使用第一阶段

按专题顺序学习。

每个专题的流程：

1. 先阅读手册：建立模型认识、记住触发条件、理解模板结构
2. 做对应训练题：以 Easy 为主，Medium 为核心，少量 Hard
3. 建立秒判能力：看到题后 5~10 秒内判断模型
4. 做复盘：分析第一反应对错、犹豫点、触发关键词

# 七、如何使用每日训练提示词

每日训练提示词是整个体系的执行器，负责出题、控制节奏、Review、复盘和模型混合训练。

推荐每日训练时长：

- 判断热身：10~15 分钟
- 主体训练：1.5~3 小时
- 复盘：20~30 分钟

推荐每天坚持训练。即使时间少，也要保持连续性。

# 八、什么时候进入第二阶段

满足以下条件后进入：

- 高频模型基本稳定（滑窗、BFS、DFS、二分、Heap、DP 等不会频繁写崩）
- Easy 题基本能独立稳定完成
- Medium 题开始出现“会做但模型犹豫”的情况

# 九、如何使用第二阶段

第二阶段不是重新刷题，而是重构认知。

重点学习：

- 为什么这些题本质相同
- 为什么这个模型能解决
- 信息流如何传播
- 状态如何维护

最终目标是识别不同题目背后的同一种结构。

# 十、关于错题

重点收藏“错误原因”，而不是大量收藏错题。

常见错误分类：

- 模型误判（例如应该 BFS 却用了 DFS）
- 边界错误（例如漏掉空数组）
- 状态错误（例如 visited 恢复失败）
- 单调性错误（例如 while 写成 if）
- DP 定义错误（状态含义不清）

# 十一、关于 Hard 题

本体系不以 Hard 题为核心。真实面试主战场是高频 Easy 和 Medium。

Hard 题用于扩展视野、深化理解和提升结构抽象能力。

# 十二、这套体系真正训练的能力

最终训练的是“模型级条件反射”：

看到关键词 → 自动触发模型

例如：

- 连续子数组 → 滑窗 / 前缀和
- 最短路径 → BFS / Dijkstra
- 所有方案 → 回溯
- 下一个更大 → 单调栈
- 层级扩散 → BFS
- 左右子树 → Tree DFS
- 区间最值 → 单调队列

# 十三、最终目标

看到题目 → 快速识别模型 → 稳定实现 → 边界不崩 → 清楚解释 → 通过真实面试

# 十四、最后的建议

不要焦虑今天刷了多少题。真正重要的是你是否正在建立：

- 模型识别能力
- 稳定实现能力
- 结构迁移能力

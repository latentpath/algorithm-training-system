# 第七章：状态转移结构（DP）

> 核心信念：**DP 不是记忆公式，而是在重叠子问题中定义状态、维护依赖并完成转移。**

---

## ① 本章解决什么结构问题？

本章解决的是：

> 当问题可以被拆成阶段，并且不同决策路径会反复到达相同子问题时，如何用状态表达子问题，并按依赖顺序求解。

DP 的核心不是“最长、最短、方案数”这些词，而是：

```text
当前状态 = 若干更小状态的组合结果
```

使用 DP 前要先回答：

- 状态表示什么？
- 状态之间如何转移？
- 初始状态是什么？
- 遍历顺序如何保证依赖已经计算？

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| 前 `i` 个元素、到达第 `i` 位 | 线性 DP |
| 两个字符串或两个序列共同推进 | 二维 DP |
| 子序列、选择或不选择 | 选择型 DP |
| 切分点、区间左右端点 | 分割 / 区间 DP |
| 方案数、最值、可行性 | 计数 / 最优 / 布尔 DP |
| 有限容量、每个物品选或不选 | 背包 DP |
| 集合状态较小 | 状态压缩 DP |

这些信号只是入口。关键仍是能否定义不重复、不遗漏的状态。

---

## ③ 本章的结构骨架

DP 通用流程：

```text
1. 定义 dp 状态含义
2. 写出转移来源
3. 确定 base case
4. 确定遍历顺序
5. 判断是否可空间优化
```

骨架：

```java
for (state in validOrder) {
    for (choice in choices) {
        dp[state] = combine(dp[state], dp[previousState]);
    }
}
```

不变量：

> 处理当前状态时，所有它依赖的前置状态都已经被正确计算。

---

## ④ 核心模型拆解

### 1）线性 DP：沿单一阶段推进

#### 结构定义

线性 DP 的状态通常表示到达某个位置、处理前 `i` 个元素或以 `i` 结尾的最优值。

#### 不变量

- `dp[i]` 的含义固定，不能一会儿表示前缀，一会儿表示以 `i` 结尾。
- 转移只依赖更早位置。
- 遍历从小到大保证依赖已计算。

#### 代码骨架

```java
int[] dp = new int[n];
dp[0] = base0;

for (int i = 1; i < n; i++) {
    dp[i] = transition(dp, i);
}
```

空间优化：

```java
int prev2 = 0;
int prev1 = 0;

for (int x : nums) {
    int curr = Math.max(prev1, prev2 + x);
    prev2 = prev1;
    prev1 = curr;
}
```

#### 适用条件

- 阶段按下标自然推进。
- 当前状态依赖有限个历史状态。
- 决策只影响之后阶段。

#### 失效条件

- 状态需要表达区间两端，应转向区间 DP。
- 选择依赖复杂集合，应转向状态压缩或搜索。
- 存在单调性可直接贪心或二分时，DP 可能过重。

---

### 2）二维 DP：两个维度共同定义状态

#### 结构定义

二维 DP 用 `dp[i][j]` 表示两个维度的子问题，例如两个字符串的前缀、区间左右端点或两个资源限制。

#### 不变量

- `dp[i][j]` 必须对应明确子问题。
- 依赖方向决定遍历顺序。
- 下标偏移要与空前缀一致。

#### 代码骨架

```java
int[][] dp = new int[m + 1][n + 1];

for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        if (A.charAt(i - 1) == B.charAt(j - 1)) {
            dp[i][j] = dp[i - 1][j - 1] + 1;
        } else {
            dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
        }
    }
}
```

#### 适用条件

- 子问题由两个独立维度共同决定。
- 转移来自左、上、左上或其他已计算区域。

#### 失效条件

- 两个维度并非状态必要条件，可以降维。
- 目标不是状态转移，而是区间表达，前缀可能更合适。

---

### 3）选择型 DP：选与不选的状态分离

#### 结构定义

选择型 DP 处理“当前元素选或不选”对后续约束的影响。

典型状态：

```text
dp[i][0] = 处理到 i，且不选 i 的最优值
dp[i][1] = 处理到 i，且选 i 的最优值
```

#### 代码骨架

```java
for (int i = 1; i < n; i++) {
    dp[i][0] = Math.max(dp[i - 1][0], dp[i - 1][1]);
    dp[i][1] = dp[i - 1][0] + value[i];
}
```

#### 适用条件

- 当前选择会限制相邻或后续选择。
- 用有限状态就能表达历史影响。

#### 失效条件

- 历史影响不能被有限状态压缩。
- 要枚举所有具体方案，应转向回溯。

---

### 4）分割与区间 DP：枚举切分点

#### 结构定义

分割 DP 中，当前状态由某个切分点 `k` 连接左右子问题。

区间 DP 常用：

```text
dp[i][j] = 区间 [i, j] 的最优值
```

#### 不变量

- 小区间先于大区间计算。
- 对每个 `[i, j]`，枚举所有合法切分点。

#### 代码骨架

```java
for (int len = 2; len <= n; len++) {
    for (int i = 0; i + len - 1 < n; i++) {
        int j = i + len - 1;
        for (int k = i; k < j; k++) {
            dp[i][j] = Math.min(
                dp[i][j],
                dp[i][k] + dp[k + 1][j] + cost(i, k, j)
            );
        }
    }
}
```

#### 适用条件

- 问题可按切分点拆成左右子问题。
- 区间内部结构决定状态。
- 合并成本可由左右状态和边界计算。

#### 失效条件

- 切分后左右并不独立。
- 问题只是区间统计，应使用前缀或扫描线。

---

### 5）背包与计数 DP：容量维度上的状态维护

#### 结构定义

背包 DP 用容量或目标和作为状态维度，处理物品选择。

0-1 背包中，每个物品最多使用一次，因此容量需要倒序遍历。

#### 代码骨架

```java
int[] dp = new int[capacity + 1];

for (int i = 0; i < n; i++) {
    int weight = weights[i];
    int value = values[i];

    for (int c = capacity; c >= weight; c--) {
        dp[c] = Math.max(dp[c], dp[c - weight] + value);
    }
}
```

#### 不变量

- 倒序遍历保证当前物品不会被重复使用。
- `dp[c]` 表示在容量 `c` 下的最优值或方案数。

#### 适用条件

- 有容量、目标和、可选物品。
- 每个物品的使用次数规则明确。

#### 失效条件

- 物品顺序本身重要，可能变成排列计数。
- 状态空间过大，需要数学或贪心优化。

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| DP vs 前缀 | 状态转移 | 区间表达 | 前缀没有决策依赖 |
| DP vs 贪心 | 保留多个状态 | 局部选择可证明全局最优 | 贪心需要交换论证 |
| DP vs 回溯 | 合并重复子问题 | 枚举选择树 | 回溯保留路径，DP 合并状态 |
| DP vs 二分 | 求状态最优值 | 定位单调临界点 | 二分常把 DP/贪心放进 `check` |
| 线性 DP vs 树形 DP | 下标阶段 | 子树状态 | 树形 DP 的依赖由树结构给出 |

---

## ⑥ 本章最容易误判的地方

- `dp[i]` 含义不稳定。
- 初始化与状态语义不一致。
- 遍历顺序没有满足依赖关系。
- 0-1 背包容量正序遍历，导致重复使用物品。
- 把子数组问题误判为子序列 DP。
- 能贪心的问题强行 DP，但没有必要状态。
- 只写转移式，不解释状态为什么覆盖所有情况。

---

## ⑦ 小结

DP 的统一视角是：

> 用状态压缩历史影响，用转移表达子问题依赖，用遍历顺序保证不变量。

真正的 DP 建模能力不在于记住题型，而在于能清楚说出：`dp[...]` 表示哪个子问题，以及为什么这些状态足以承载未来决策。

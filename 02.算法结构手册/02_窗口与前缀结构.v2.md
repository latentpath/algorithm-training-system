# 第二章：窗口与前缀结构

> 核心信念：**窗口解决连续区间的局部维护，前缀解决区间信息的全局表达。**

---

## ① 本章解决什么结构问题？

本章解决的是连续区间问题中的两类建模方式：

- 窗口结构：动态维护当前连续区间 `[left, right]` 的状态。
- 前缀结构：用两个累积状态的差表达任意区间。

二者都处理“区间”，但视角不同：

| 结构 | 核心问题 | 状态位置 |
|---|---|---|
| 窗口 | 当前区间能否增量维护并安全收缩 | 局部动态状态 |
| 前缀 | 任意区间如何转化为两个全局状态的关系 | 全局静态表达 |

窗口成立的关键是收缩安全。当前缀成立的关键是区间信息可以被累积状态表达。

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 优先考虑 |
|---|---|
| 连续子数组 / 子串，维护当前区间统计 | 窗口 |
| 最长 / 最短连续区间，约束可随边界单调变化 | 窗口 |
| 任意起点终点的区间和 / 区间计数 | 前缀 |
| 存在负数导致窗口和不再单调 | 前缀 + 哈希 / 单调队列 |
| 恰好为 K 的区间数量 | 前缀 + 哈希 |
| 区间批量加减，最后查询结果 | 差分 |

不要只因为出现“连续”就使用窗口。必须判断：当窗口不合法时，移动 `left` 是否一定不会错过答案。

---

## ③ 本章的结构骨架

### 窗口骨架

```text
[left, right]  当前窗口
state          窗口内统计量
```

```java
int left = 0;

for (int right = 0; right < n; right++) {
    add(nums[right]);

    while (!valid()) {
        remove(nums[left]);
        left++;
    }

    updateAnswer(left, right);
}
```

### 前缀骨架

```text
prefix[i] = nums[0] + ... + nums[i - 1]
sum(l, r) = prefix[r + 1] - prefix[l]
```

```java
int prefix = 0;
Map<Integer, Integer> count = new HashMap<>();
count.put(0, 1);

for (int num : nums) {
    prefix += num;
    answer += count.getOrDefault(prefix - k, 0);
    count.put(prefix, count.getOrDefault(prefix, 0) + 1);
}
```

窗口强调“维护当前状态”，前缀强调“表达区间关系”。

## ④ 核心模型拆解

### 1）变长窗口：连续区间的局部维护

#### 结构定义

变长窗口用两个边界维护一个连续区间，并在扩张、收缩之间保持窗口合法。

```text
[0, left)       不再作为当前窗口起点
[left, right]   当前窗口
(right, n - 1]  未进入窗口
```

#### 不变量

- `state` 精确描述 `[left, right]`。
- 当 `valid()` 为真时，当前窗口满足约束。
- 当 `valid()` 为假时，收缩 `left` 能向合法状态推进。

#### 代码骨架

```java
int left = 0;

for (int right = 0; right < n; right++) {
    add(nums[right]);

    while (!valid()) {
        remove(nums[left]);
        left++;
    }

    updateAnswer();
}
```

#### 适用条件

- 目标对象是连续子数组或子串。
- 加入右端、移除左端都能增量维护状态。
- 约束具有足够单调性，收缩不会错过未来答案。

#### 失效条件

- 区间和中存在负数，导致扩大或收缩不再单调。
- 约束依赖全局选择，不能靠窗口局部状态维护。
- 目标是子序列，不是连续区间。

失效时转向前缀和、哈希、单调队列、DP 或搜索。

---

### 2）定长窗口：固定跨度的滚动状态

#### 结构定义

定长窗口维护长度固定为 `k` 的连续区间。

#### 不变量

处理到 `right` 时，窗口包含最近 `k` 个元素：

```text
[right - k + 1, right]
```

当 `right < k - 1` 时，窗口尚未形成。

#### 代码骨架

```java
for (int right = 0; right < n; right++) {
    add(nums[right]);

    if (right >= k) {
        remove(nums[right - k]);
    }

    if (right >= k - 1) {
        updateAnswer();
    }
}
```

#### 适用条件

- 区间长度固定。
- 每次移动只新增一个元素、移除一个元素。
- 状态可增量更新。

#### 失效条件

- 长度不是固定约束，而是由合法性动态决定。
- 需要窗口最大值 / 最小值时，普通变量无法处理过期元素，应使用单调队列。

---

### 3）前缀和：用两个全局状态表达区间

#### 结构定义

前缀结构将区间问题转换为两个累积状态的差：

```text
prefix[j] - prefix[i] = nums[i] + ... + nums[j - 1]
```

它不维护当前窗口，而是把所有可能区间映射到前缀状态之间的关系。

#### 不变量

- `prefix` 表示扫描到当前位置前的累计状态。
- 哈希表记录已出现的前缀状态及次数。
- 对当前位置 `j`，所有满足条件的起点 `i` 都在历史前缀中查询。

#### 代码骨架

```java
Map<Integer, Integer> count = new HashMap<>();
count.put(0, 1);

int prefix = 0;
int answer = 0;

for (int num : nums) {
    prefix += num;
    answer += count.getOrDefault(prefix - k, 0);
    count.put(prefix, count.getOrDefault(prefix, 0) + 1);
}
```

#### 适用条件

- 区间信息可由两个前缀状态组合得到。
- 需要统计任意起点和终点。
- 数据存在负数，窗口无法安全收缩。

#### 失效条件

- 状态不是可逆或可差分的累积表达。
- 区间约束依赖内部顺序或局部结构。
- 需要的是最优决策转移而非区间表达，应转向 DP。

---

### 4）前缀最值与单调队列

#### 结构定义

有些问题不是查询某个前缀值是否存在，而是在历史前缀中找最优候选。例如最短子数组和至少为 `K`：

```text
prefix[j] - prefix[i] >= K
```

此时需要维护可能成为最优起点的前缀集合。

#### 不变量

单调队列中保存前缀下标，且对应前缀值单调递增：

```text
prefix[dq[0]] <= prefix[dq[1]] <= ...
```

较大的早期前缀如果被更小的新前缀支配，就可以删除。

#### 代码骨架

```java
Deque<Integer> dq = new ArrayDeque<>();

for (int j = 0; j <= n; j++) {
    while (!dq.isEmpty() && prefix[j] - prefix[dq.peekFirst()] >= k) {
        answer = Math.min(answer, j - dq.pollFirst());
    }

    while (!dq.isEmpty() && prefix[dq.peekLast()] >= prefix[j]) {
        dq.pollLast();
    }

    dq.offerLast(j);
}
```

#### 适用条件

- 前缀候选存在支配关系。
- 需要最短、最优或临界区间。
- 普通哈希无法表达最优顺序。

---

### 5）差分结构：区间更新的前缀反向表达

#### 结构定义

差分把“区间加”转化为两个边界事件：

```text
diff[i] = nums[i] - nums[i - 1]
```

区间 `[l, r]` 加 `val`：

```java
diff[l] += val;
diff[r + 1] -= val;
```

最后对 `diff` 做前缀还原。

#### 适用条件

- 存在大量区间更新。
- 查询可以延后统一处理。
- 更新是可累积的加减操作。

#### 失效条件

- 更新后需要立即查询复杂统计。
- 操作不是可累积的线性变化。
- 区间关系需要排序或重叠判断，应转向区间结构。

---

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| 窗口 vs 指针 | 窗口维护连续区间统计 | 指针泛指边界移动 | 窗口必须有可增量维护的 `state` |
| 窗口 vs 前缀 | 动态维护局部区间 | 静态表达任意区间 | 窗口要求收缩安全，前缀不要求 |
| 前缀 vs DP | 区间表达 | 状态转移 | 前缀没有决策依赖 |
| 前缀 vs 二分 | 累积关系查询 | 临界点定位 | 二分依赖判定单调性 |
| 差分 vs 扫描线 | 区间批量更新 | 事件顺序统计 | 扫描线关注事件发生顺序和覆盖状态 |

---

## ⑥ 本章最容易误判的地方

- 存在负数时仍用和窗口求最短区间。
- 把子序列误判为子数组窗口。
- 忘记 `map.put(0, 1)`，导致从下标 0 开始的区间丢失。
- 在窗口收缩前后更新答案的位置不符合题意。
- 需要窗口极值时用普通变量，无法处理过期元素。
- 差分更新时忘记 `r + 1` 的边界。

---

## ⑦ 小结

窗口与前缀的统一视角是：

> 都处理连续区间，但窗口维护当前区间，前缀表达任意区间。

当窗口无法证明收缩安全时，通常应提升到前缀视角；当前缀只负责表达而无法完成决策时，再考虑 DP、单调队列或二分。

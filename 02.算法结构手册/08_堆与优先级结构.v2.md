# 第八章：堆与优先级结构

> 核心信念：**当问题的核心变成“下一步处理谁”时，优先级结构负责维护当前最优候选。**

---

## ① 本章解决什么结构问题？

本章解决的是：

> 在一组动态变化的候选元素中，如何高效获得当前优先级最高或最低的元素。

堆不是“排序的替代品”。排序适合一次性全局有序；堆适合动态加入、删除和反复取最优。

常见任务：

- 维护 Top K。
- 合并多条有序流。
- 数据流实时最值或中位数。
- 图算法中优先扩展当前最优状态。
- 任务调度中按优先级选择下一个任务。

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| Top K、第 K 大 / 小 | 固定大小堆 |
| 数据持续加入，需要实时最值 | 数据流堆 |
| 合并 K 个有序序列 | 多路合并堆 |
| 每次选择当前最小代价状态 | Dijkstra / 优先搜索 |
| 同时维护较小一半和较大一半 | 双堆 |
| 任务冷却、会议室、调度 | 优先级调度 |

判断关键是：候选集合是否动态变化，且每一步是否只需要当前最优候选。

---

## ③ 本章的结构骨架

Java 中常用 `PriorityQueue`：

```java
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
PriorityQueue<Integer> maxHeap =
    new PriorityQueue<>((a, b) -> Integer.compare(b, a));
```

核心操作：

| 操作 | 含义 | 复杂度 |
|---|---|---|
| `offer` | 插入候选 | `O(log n)` |
| `poll` | 取出堆顶 | `O(log n)` |
| `peek` | 查看堆顶 | `O(1)` |

不变量：

> 堆顶始终是当前候选集合中优先级最高的元素。

---

## ④ 核心模型拆解

### 1）Top K：固定大小候选集合

#### 结构定义

Top K 结构维护当前最好的 K 个元素。若要保留最大的 K 个，通常使用大小为 K 的最小堆，堆顶是这 K 个中最弱的候选。

#### 不变量

- 堆大小不超过 `k`。
- 扫描到当前位置时，堆内是已扫描元素中的 Top K。
- 堆顶是当前 Top K 的边界。

#### 代码骨架

```java
PriorityQueue<Integer> heap = new PriorityQueue<>();

for (int num : nums) {
    heap.offer(num);
    if (heap.size() > k) {
        heap.poll();
    }
}

// heap 中保留最大的 k 个元素
```

#### 适用条件

- 只关心前 K 个，不需要完整排序。
- 数据可一次扫描或流式到达。
- K 明显小于 N 时收益更明显。

#### 失效条件

- 需要完整有序输出，排序更直接。
- 需要频繁删除任意元素，普通堆不支持 `O(log n)` 定位删除。
- K 接近 N 且只处理一次，排序可能更简单。

---

### 2）多路合并：堆维护每条有序流的头部

#### 结构定义

多路合并中，每条序列内部有序。堆中只放每条序列当前头部候选，每次取全局最小后推进对应序列。

#### 不变量

- 堆中每个元素代表一条来源的当前未消费头部。
- 已输出结果保持有序。
- 某来源被取出后，只需要放入该来源的下一个元素。

#### 代码骨架

```java
PriorityQueue<int[]> pq =
    new PriorityQueue<>((a, b) -> Integer.compare(a[0], b[0]));

for (int row = 0; row < k; row++) {
    if (matrix[row].length > 0) {
        pq.offer(new int[]{matrix[row][0], row, 0});
    }
}

while (!pq.isEmpty()) {
    int[] curr = pq.poll();
    int value = curr[0];
    int row = curr[1];
    int index = curr[2];

    take(value);

    if (index + 1 < matrix[row].length) {
        pq.offer(new int[]{matrix[row][index + 1], row, index + 1});
    }
}
```

#### 适用条件

- 每个来源内部有序。
- 来源数量较多，手写多指针比较不稳定。
- 每次只需要全局下一个最优元素。

#### 失效条件

- 来源内部无序。
- 只有两个来源时，双指针合并更简单。
- 需要随机访问和删除任意候选，堆能力不够。

---

### 3）双堆：动态维护中位数或平衡分区

#### 结构定义

双堆用一个最大堆维护较小一半，一个最小堆维护较大一半。

```text
small: 较小一半，堆顶是较小一半最大值
large: 较大一半，堆顶是较大一半最小值
```

#### 不变量

- `small.size()` 与 `large.size()` 的差不超过 1。
- `small` 中所有元素小于等于 `large` 中所有元素。
- 中位数可由一个或两个堆顶得到。

#### 代码骨架

```java
PriorityQueue<Integer> small =
    new PriorityQueue<>((a, b) -> Integer.compare(b, a));
PriorityQueue<Integer> large = new PriorityQueue<>();

void addNum(int num) {
    small.offer(num);
    large.offer(small.poll());

    if (large.size() > small.size()) {
        small.offer(large.poll());
    }
}
```

#### 适用条件

- 数据流动态加入。
- 需要实时中位数或动态平衡分区。

#### 失效条件

- 需要删除任意历史元素，普通双堆需要延迟删除配合哈希。
- 需要分位数而非中位数，结构要重新设计。

---

### 4）堆 + 图：优先扩展当前最优状态

#### 结构定义

在 Dijkstra、Prim 或 A* 中，堆维护待扩展状态的优先级。

#### 不变量

- 堆顶是当前最有希望或当前代价最小的状态。
- 出堆状态若已过期，需要跳过。
- 每次松弛后把更优状态重新加入堆。

#### 代码骨架

```java
PriorityQueue<int[]> pq =
    new PriorityQueue<>((a, b) -> Integer.compare(a[1], b[1]));

pq.offer(new int[]{start, 0});

while (!pq.isEmpty()) {
    int[] state = pq.poll();
    if (isExpired(state)) continue;

    for (State next : expand(state)) {
        if (relax(next)) {
            pq.offer(encode(next));
        }
    }
}
```

#### 适用条件

- 扩展顺序影响效率或正确性。
- 当前最小代价状态可以稳定推进。

#### 失效条件

- 边权为负，Dijkstra 的稳定性不成立。
- 优先级不是全局可比较的。

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| 堆 vs 排序 | 动态维护优先级 | 一次性全局有序 | 需要反复取最优时用堆 |
| 堆 vs 单调队列 | 全局候选优先级 | 滑动窗口局部极值 | 单调队列能处理窗口过期顺序 |
| 堆 vs 双指针合并 | 多来源动态最优 | 少量来源线性推进 | 两路合并不需要堆 |
| 堆 vs BST / TreeMap | 快速取堆顶 | 有序查询、任意删除 | 普通堆不擅长查找和删除任意元素 |
| 堆 vs 图最短路 | 堆是组件 | 图算法是结构 | 堆只负责选择下一个状态 |

---

## ⑥ 本章最容易误判的地方

- 最大 K 个却使用最大堆，导致堆大小边界难维护。
- 忘记限制堆大小。
- Comparator 写反或用减法导致溢出。
- 用堆处理滑动窗口删除，却没有处理过期元素。
- 用堆替代简单排序，使实现复杂但收益不大。
- 在 Dijkstra 中不跳过过期状态。

---

## ⑦ 小结

堆与优先级结构的统一视角是：

> 动态候选集合中，堆负责稳定提供当前最优候选。

当问题反复问“下一步选谁”，且候选集合会变化时，优先级结构通常成立；如果只需要一次性有序，排序往往更简单。

# 第十章：单调结构（Java）

> 核心信念：**单调结构通过维护支配关系，延迟处理仍可能成为答案的元素。**

---

## ① 本章解决什么结构问题？

本章解决的是：

> 在线性扫描中，如何用单调栈或单调队列维护“尚未被支配”的候选元素，从而在 `O(n)` 时间内定位最近关系或窗口极值。

单调结构不是普通栈和队列的使用技巧，而是利用支配关系删除不可能成为答案的元素。

常见任务：

- 下一个更大 / 更小元素。
- 左边或右边第一个更大 / 更小元素。
- 柱状图最大矩形。
- 每个元素作为区间最值的贡献。
- 滑动窗口最大值 / 最小值。

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| 最近更大 / 最近更小 | 单调栈 |
| 下一个更大元素 | 单调栈 |
| 每个元素作为最值的影响范围 | 单调栈贡献法 |
| 滑动窗口最大值 / 最小值 | 单调队列 |
| 暴力需要向左或向右找第一个支配元素 | 单调栈 |
| 候选会过期且需维护极值 | 单调队列 |

如果问题只需要全局最大值，普通变量即可；如果需要动态全局优先级，堆更合适。

---

## ③ 本章的结构骨架

单调结构维护两类不变量：

```text
单调栈：栈内元素按值保持单调，表示尚未找到支配者的候选
单调队列：队列内元素按值单调，同时按下标保持窗口顺序
```

Java 推荐使用：

```java
Deque<Integer> stack = new ArrayDeque<>();
Deque<Integer> deque = new ArrayDeque<>();
```

通常存下标而不是值，因为下标能同时表达位置、过期和距离。

---

## ④ 核心模型拆解

### 1）单调栈：寻找最近支配关系

#### 结构定义

单调栈用于寻找某个方向上第一个更大或更小的元素。栈中保存尚未找到答案的候选。

#### 不变量

以“下一个更大元素”为例：

- 栈中下标对应的值保持单调递减。
- 当前元素 `nums[i]` 如果大于栈顶，就成为栈顶元素的下一个更大值。
- 每个元素最多入栈一次、出栈一次。

#### 代码骨架

```java
int[] res = new int[n];
Arrays.fill(res, -1);
Deque<Integer> stack = new ArrayDeque<>();

for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && nums[stack.peek()] < nums[i]) {
        int index = stack.pop();
        res[index] = nums[i];
    }
    stack.push(i);
}
```

#### 适用条件

- 需要找最近的更大 / 更小关系。
- 暴力会对每个元素向一侧扫描。
- 元素之间存在明确支配关系。

#### 失效条件

- 目标不是最近关系，而是全局 Top K，应使用堆。
- 关系不具备单调支配性。
- 需要维护可删除任意元素的有序集合，应考虑 TreeMap。

---

### 2）反向扫描：从右侧寻找候选

#### 结构定义

有些问题要找右侧第一个更大元素，可以从右向左扫描，让栈表示当前位置右侧的候选集合。

#### 不变量

- 扫描到 `i` 时，栈内元素都来自 `i` 的右侧。
- 被当前元素支配的右侧候选可以删除。

#### 代码骨架

```java
int[] res = new int[n];
Deque<Integer> stack = new ArrayDeque<>();

for (int i = n - 1; i >= 0; i--) {
    while (!stack.isEmpty() && stack.peek() <= nums[i]) {
        stack.pop();
    }

    res[i] = stack.isEmpty() ? -1 : stack.peek();
    stack.push(nums[i]);
}
```

#### 适用条件

- 答案来自右侧候选。
- 不需要候选下标或距离时可以存值。

#### 失效条件

- 需要距离、过期或重复值边界时，应存下标。

---

### 3）贡献法：确定元素作为最值的控制范围

#### 结构定义

贡献法不是直接找答案，而是计算每个元素作为区间最小值或最大值时能覆盖多少区间。

核心是找到左右边界：

```text
left[i]  = 左侧第一个严格更小 / 更大位置
right[i] = 右侧第一个小于等于 / 大于等于位置
```

相等条件必须一侧严格、一侧非严格，用来避免重复计数。

#### 代码骨架

```java
int[] left = new int[n];
int[] right = new int[n];
Deque<Integer> stack = new ArrayDeque<>();

for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && nums[stack.peek()] > nums[i]) {
        stack.pop();
    }
    left[i] = stack.isEmpty() ? i + 1 : i - stack.peek();
    stack.push(i);
}

stack.clear();

for (int i = n - 1; i >= 0; i--) {
    while (!stack.isEmpty() && nums[stack.peek()] >= nums[i]) {
        stack.pop();
    }
    right[i] = stack.isEmpty() ? n - i : stack.peek() - i;
    stack.push(i);
}
```

#### 适用条件

- 答案可以拆成每个元素的贡献。
- 需要知道一个元素作为区间最值的影响范围。

#### 失效条件

- 区间贡献不是由单个最值决定。
- 相等元素的归属规则无法明确。

---

### 4）单调队列：滑动窗口内的局部极值

#### 结构定义

单调队列维护当前窗口内可能成为最大值或最小值的下标。它同时处理两个约束：

- 值的单调性。
- 下标是否过期。

#### 不变量

以窗口最大值为例：

- 队列下标递增。
- 队列对应值递减。
- 队首始终是当前窗口最大值下标。

#### 代码骨架

```java
Deque<Integer> deque = new ArrayDeque<>();

for (int i = 0; i < n; i++) {
    while (!deque.isEmpty() && nums[deque.peekLast()] <= nums[i]) {
        deque.pollLast();
    }

    deque.offerLast(i);

    if (deque.peekFirst() <= i - k) {
        deque.pollFirst();
    }

    if (i >= k - 1) {
        result[i - k + 1] = nums[deque.peekFirst()];
    }
}
```

#### 适用条件

- 目标是滑动窗口最大值或最小值。
- 窗口按时间顺序移动，旧元素会过期。
- 每次只需当前窗口极值。

#### 失效条件

- 需要全局动态优先级且没有窗口过期顺序，应使用堆。
- 需要任意删除或有序查询，应使用 TreeMap。

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| 单调栈 vs 堆 | 最近支配关系 | 全局优先级 | 单调栈按线性位置消解 |
| 单调队列 vs 堆 | 窗口局部极值 | 动态全局最值 | 单调队列天然处理过期顺序 |
| 单调栈 vs 双指针 | 候选支配删除 | 边界移动 | 单调栈保存历史候选 |
| 单调结构 vs 区间结构 | 原序线性扫描 | 排序后扫描区间 | 单调结构通常不重排输入 |
| 贡献法 vs DP | 元素独立贡献 | 状态转移依赖 | 贡献法没有阶段决策 |

---

## ⑥ 本章最容易误判的地方

- 栈中存值还是下标混乱。
- `<`、`<=` 使用错误，导致重复值计数错。
- 单调队列忘记弹出过期下标。
- 把全局最值问题误用单调队列。
- 把没有最近支配关系的问题强行套单调栈。
- 贡献法中左右边界的严格性不配套。

---

## ⑦ 小结

单调结构的统一视角是：

> 用单调性删除被支配候选，只保留未来仍可能成为答案的元素。

如果一个元素在当前扫描位置已经确定不可能再成为答案，就可以出栈或出队。每个元素只进出一次，因此线性复杂度来自结构不变量，而不是模板本身。

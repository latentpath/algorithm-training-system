# 第四章：二分结构

> 核心信念：**二分的本质不是在数组里找数，而是在单调结构中寻找临界点。**

---

## ① 本章解决什么结构问题？

本章解决的是：

> 在一个可比较、可判定且具有单调性的空间中，寻找边界位置。

二分结构的核心不是“有序数组”，而是存在一个临界点：

```text
false false false true true true
                  ^
                boundary
```

或：

```text
true true true false false false
              ^
            boundary
```

二分能用的前提是：`check(x)` 的真假随 `x` 单调变化。

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| 最小满足某条件 | 寻找第一个 `true` |
| 最大满足某条件 | 寻找最后一个 `true` |
| `capacity / speed / time / limit` | 答案二分 |
| `minimize the maximum` | 最小可行值 |
| `maximize the minimum` | 最大可行值 |
| 有序数组中找边界 | lower_bound / upper_bound |
| “是否可以完成” | 判定函数模型 |

触发词不能替代证明。使用二分前必须说明：为什么 `check(x)` 单调。

---

## ③ 本章的结构骨架

### 寻找最小满足条件的值

```java
int left = low;
int right = high;

while (left < right) {
    int mid = left + (right - left) / 2;
    if (check(mid)) {
        right = mid;
    } else {
        left = mid + 1;
    }
}

return left;
```

不变量：

```text
答案一定在 [left, right]
right 侧保留一个可行候选
left 侧排除已证明不可行的值
```

### 寻找最大满足条件的值

```java
int left = low;
int right = high;

while (left < right) {
    int mid = left + (right - left + 1) / 2;
    if (check(mid)) {
        left = mid;
    } else {
        right = mid - 1;
    }
}

return left;
```

向右收缩时用偏右 `mid`，避免 `left == mid` 死循环。

---

## ④ 核心模型拆解

### 1）有序数组查找：位置空间上的二分

#### 结构定义

在有序数组中，目标位置左右两侧具有可比较关系。

#### 不变量

```text
[left, right] 表示仍可能包含 target 的闭区间
```

每次比较 `nums[mid]` 与 `target`，排除不可能的一半。

#### 代码骨架

```java
int left = 0;
int right = nums.length - 1;

while (left <= right) {
    int mid = left + (right - left) / 2;

    if (nums[mid] == target) {
        return mid;
    } else if (nums[mid] < target) {
        left = mid + 1;
    } else {
        right = mid - 1;
    }
}

return -1;
```

#### 适用条件

- 数据按可比较维度有序。
- 比较结果能排除一半候选。

#### 失效条件

- 数组无序。
- 要找的不是位置，而是满足条件的边界，应转向判定函数二分。

---

### 2）边界二分：lower_bound / upper_bound

#### 结构定义

边界二分不是找某个值，而是找第一个或最后一个满足关系的位置。

例如第一个 `>= target` 的位置：

```text
< target | >= target
          ^
```

#### 代码骨架

```java
int left = 0;
int right = nums.length;

while (left < right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] >= target) {
        right = mid;
    } else {
        left = mid + 1;
    }
}

return left;
```

#### 适用条件

- 目标是插入位置、左右边界或计数边界。
- 条件随下标单调。

#### 失效条件

- 需要所有匹配位置且数据无序。
- 相等元素的边界语义未定义。

---

### 3）答案二分：在值域上寻找临界点

#### 结构定义

答案二分把优化问题转化为判定问题：

```text
check(x): 在限制 x 下是否可行？
```

如果 `x` 可行时更大的 `x` 也可行，就能找最小可行值。

#### 不变量

- `[left, right]` 覆盖所有可能答案。
- `check(mid)` 为真时，答案不大于 `mid`。
- `check(mid)` 为假时，答案大于 `mid`。

#### 代码骨架

```java
int left = minPossible;
int right = maxPossible;

while (left < right) {
    int mid = left + (right - left) / 2;
    if (canFinish(mid)) {
        right = mid;
    } else {
        left = mid + 1;
    }
}

return left;
```

#### 适用条件

- 能定义清晰的答案上下界。
- 能写出可行性判定函数。
- 判定函数具有单调性。
- `O(log range * checkCost)` 可接受。

#### 失效条件

- 判定函数不单调。
- 无法确定答案边界。
- `check` 本身需要求最优且复杂度过高。
- 目标可直接贪心或 DP 求出，无需二分外壳。

---

### 4）单调性来源

常见单调性：

| 限制变量 | 单调方向 |
|---|---|
| 容量越大 | 越容易完成 |
| 时间越多 | 越容易完成 |
| 速度越快 | 越容易完成 |
| 最大允许值越大 | 约束越宽 |
| 最小间距越大 | 越难满足 |

结构判断时要把“变量增加后更容易还是更难”说清楚。

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| 二分 vs 对撞指针 | 单调判定空间折半 | 两端线性收缩 | 二分依赖全局单调性 |
| 二分 vs 贪心 | 二分定位临界值 | 贪心直接构造答案 | 答案二分常用贪心写 `check` |
| 二分 vs DP | 判定可行性 | 状态转移求最优 | DP 负责状态依赖，二分负责值域边界 |
| 二分 vs 单调栈 | 找临界点 | 找最近支配关系 | 单调栈处理线性邻近关系 |

---

## ⑥ 本章最容易误判的地方

- `check` 不单调，却强行二分。
- `high` 设得过小，答案不在搜索区间内。
- 寻找最大满足值时使用偏左 `mid` 导致死循环。
- `left <= right` 与 `left < right` 混用，边界语义不一致。
- 把“局部有序”误当整体有序。
- `check` 中使用贪心但没有证明贪心判定正确。

---

## ⑦ 小结

二分结构的统一视角是：

> 定义单调判定函数，并在判定空间中寻找临界点。

使用二分前只问四件事：空间是什么，边界在哪里，`check` 表示什么，为什么单调。四件事不清楚，二分代码再熟也不可靠。

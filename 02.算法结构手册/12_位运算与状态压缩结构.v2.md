# 第十二章：位运算与状态压缩结构

> 核心信念：**位运算的本质不是技巧，而是用二进制编码集合、状态和约束。**

---

## ① 本章解决什么结构问题？

本章解决的是：

> 当问题中的状态可以被离散为存在 / 不存在、访问 / 未访问、选择 / 未选择时，如何用二进制位压缩表达，并在此基础上进行搜索或 DP。

位运算不是为了写得短，而是为了建立状态模型：

| 层级 | 含义 |
|---|---|
| 单个 bit | 一个布尔状态 |
| bitmask | 一个集合状态 |
| `dp[mask]` | 以集合为维度的状态转移 |
| `(node, mask)` | 图位置 + 已访问集合 |

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| `XOR`、只出现一次 | 奇偶抵消 |
| 是否访问、是否选择 | bitmask |
| 枚举所有子集 | 子集枚举 |
| `n <= 20` 且涉及集合状态 | 状态压缩 |
| TSP、访问所有点 | `dp[mask][i]` |
| N 皇后优化 | 位集合表示列和对角线 |
| 奇偶性、parity | XOR / bit 状态 |

进入位运算前先确认：一个 bit 是否能稳定对应一个状态。

---

## ③ 本章的结构骨架

### 基础状态表达

```text
mask 的第 i 位 = 元素 i 是否存在 / 已访问 / 已选择
```

常见操作：

| 操作 | 含义 |
|---|---|
| `(mask & (1 << i)) != 0` | 判断第 `i` 位是否存在 |
| `mask | (1 << i)` | 添加元素 `i` |
| `mask & ~(1 << i)` | 删除元素 `i` |
| `mask ^ (1 << i)` | 翻转元素 `i` 状态 |
| `mask & (mask - 1)` | 删除最低位的 1 |
| `mask & -mask` | 取最低位的 1 |

### 状态压缩骨架

```java
for (int mask = 0; mask < (1 << n); mask++) {
    for (int i = 0; i < n; i++) {
        if ((mask & (1 << i)) == 0) {
            int next = mask | (1 << i);
            transition(mask, next, i);
        }
    }
}
```

不变量：

> `mask` 必须始终准确表示当前集合状态，位与元素的映射不能变化。

## ④ 核心模型拆解

### 1）XOR：奇偶抵消结构

#### 结构定义

XOR 表达的是奇偶性。相同元素出现两次会相互抵消：

```text
a ^ a = 0
a ^ 0 = a
```

#### 不变量

- 已处理元素的 XOR 值表示所有出现奇数次元素的异或结果。
- 成对出现的元素不会影响最终结果。

#### 代码骨架

```java
int ans = 0;

for (int num : nums) {
    ans ^= num;
}
```

#### 适用条件

- 问题存在成对抵消关系。
- 需要维护奇偶性或差异位。
- 操作满足交换律和结合律。

#### 失效条件

- 出现次数不是基于奇偶抵消。
- 需要保留元素顺序或频率完整信息。
- 多个不同奇数次元素无法仅靠一个 XOR 还原全部信息。

---

### 2）lowbit 与删除最低位：位级边界操作

#### 结构定义

`x & -x` 提取最低位的 1：

```text
x      = 110100
lowbit = 000100
```

`x & (x - 1)` 删除最低位的 1。

#### 代码骨架

```java
int lowbit = x & -x;

while (x != 0) {
    x &= x - 1;
    count++;
}
```

#### 适用条件

- 需要枚举 mask 中存在的位。
- 需要统计 1 的个数。
- Fenwick Tree 或子状态枚举中需要最低位操作。

#### 失效条件

- 状态不是二进制集合。
- 负数补码语义会影响理解，需明确整数范围。

---

### 3）bitmask：用整数表达集合

#### 结构定义

bitmask 把集合压缩到一个整数中：

```text
mask = 10110
表示集合 {1, 2, 4}
```

#### 不变量

- 第 `i` 位始终对应同一个元素。
- 修改集合必须通过位操作更新 `mask`。
- `mask == (1 << n) - 1` 表示所有元素都已包含。

#### 代码骨架

```java
boolean exists = (mask & (1 << i)) != 0;
int add = mask | (1 << i);
int remove = mask & ~(1 << i);
```

#### 适用条件

- 元素数量较小，通常 `n <= 20` 到 `25`。
- 状态是二元的。
- 需要快速复制、比较或缓存集合状态。

#### 失效条件

- 元素数量过大，状态空间 `2^n` 不可承受。
- 每个元素有多种状态，不是简单二元选择。
- 需要保存复杂对象集合，bitmask 只是编码不够表达全部信息。

---

### 4）子集枚举：遍历集合状态空间

#### 结构定义

长度为 `n` 的集合共有 `2^n` 个子集。每个 `mask` 对应一个子集。

#### 代码骨架

```java
for (int mask = 0; mask < (1 << n); mask++) {
    for (int i = 0; i < n; i++) {
        if ((mask & (1 << i)) != 0) {
            use(i);
        }
    }
}
```

枚举某个集合的所有子集：

```java
for (int sub = mask; sub > 0; sub = (sub - 1) & mask) {
    use(sub);
}
```

#### 适用条件

- 需要遍历所有集合状态。
- `n` 足够小。
- 每个集合状态可以独立计算或转移。

#### 失效条件

- `2^n` 规模不可接受。
- 状态存在更强结构，可用贪心、前缀或排序降低复杂度。

---

### 5）状态压缩搜索：把 visited 压成整数

#### 结构定义

在搜索中，`mask` 记录已经访问或已经选择的元素。

#### 不变量

- 当前递归层的 `mask` 表示路径中的已选择集合。
- 下一步只能选择 `mask` 中不存在的元素。
- 如果同一 `mask` 可由多条路径到达，可能需要记忆化。

#### 代码骨架

```java
void dfs(int mask) {
    if (mask == target) {
        return;
    }

    for (int i = 0; i < n; i++) {
        if ((mask & (1 << i)) != 0) continue;
        dfs(mask | (1 << i));
    }
}
```

#### 适用条件

- 搜索状态主要由已选集合决定。
- `boolean[] used` 可以被压缩为整数。
- 需要把状态作为 Map key 或 DP 下标。

#### 失效条件

- 路径顺序对未来有影响，但 mask 没有记录当前位置。
- 除集合外还有其他必要状态，应组合成 `(pos, mask)`。

---

### 6）状态压缩 DP：集合维度上的状态转移

#### 结构定义

状态压缩 DP 用集合表示高维选择状态。典型形式：

```text
dp[mask][i] = 已访问 mask 集合，当前停在 i 的最优值
```

#### 不变量

- `i` 必须属于 `mask`。
- 从 `mask` 转移到 `nextMask` 时，只添加一个未访问元素。
- 遍历顺序按 mask 扩张，确保前置状态已计算。

#### 代码骨架

```java
for (int mask = 0; mask < (1 << n); mask++) {
    for (int i = 0; i < n; i++) {
        if ((mask & (1 << i)) == 0) continue;

        for (int j = 0; j < n; j++) {
            if ((mask & (1 << j)) != 0) continue;

            int next = mask | (1 << j);
            dp[next][j] = Math.min(
                dp[next][j],
                dp[mask][i] + cost[i][j]
            );
        }
    }
}
```

#### 适用条件

- 集合状态是必要维度。
- `n` 较小。
- 未来决策依赖已访问集合和当前位置。

#### 失效条件

- 状态空间过大。
- 集合顺序影响无法由当前位置补足。
- 问题没有重叠子问题，只需回溯枚举。

---

### 7）位运算优化约束判断

#### 结构定义

在 N 皇后等问题中，可以用 bitmask 表示列、主对角线、副对角线是否被占用。

#### 不变量

- 每个 bit 表示一个冲突资源是否已被占用。
- 放置前检查冲突，放置后更新 mask，回溯时恢复。

#### 适用条件

- 约束可以映射为有限个布尔集合。
- 频繁检查合法性，需要 `O(1)` 判定。

#### 失效条件

- 约束不是二元占用关系。
- 棋盘或状态规模超过整数位宽，需要 `long`、`BitSet` 或其他结构。

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| bitmask vs HashSet | 压缩集合表达 | 通用集合容器 | bitmask 要求元素可映射到小整数 |
| 状压 DP vs 普通 DP | 集合是状态维度 | 下标或区间是状态维度 | 状压 DP 的核心是 `mask` |
| bitmask 搜索 vs 回溯 | 状态编码优化 | 搜索树结构 | 位运算不替代搜索，只压缩状态 |
| XOR vs 计数哈希 | 奇偶抵消 | 完整频率统计 | XOR 只保留奇偶信息 |
| bitmask vs Trie | 状态压缩 | 前缀共享 | 两者压缩对象不同 |

---

## ⑥ 本章最容易误判的地方

- 没有定义第 `i` 位对应什么元素。
- 位移表达式缺少括号。
- `1 << n` 溢出，应该使用 `1L << n`。
- `mask` 只记录集合，却忘记记录当前位置。
- XOR 与 OR 混淆：XOR 是翻转 / 奇偶，OR 是添加。
- 状态空间 `2^n` 过大仍强行状压。
- 相同集合但不同路径信息不可合并时误用 `dp[mask]`。

---

## ⑦ 小结

位运算与状态压缩的统一视角是：

> 用二进制位建立状态模型，再把集合状态作为搜索或 DP 的基础。

真正重要的不是记住某个公式，而是判断一个整数是否足以表达当前问题的状态空间。如果状态表达成立，位运算会让搜索、缓存和转移都更紧凑。

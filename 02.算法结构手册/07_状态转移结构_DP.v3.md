# 第七章：状态转移结构（DP）

> 核心信念：**DP 不是记忆公式，而是在重叠子问题中定义状态、维护依赖并完成转移。**

---

# ① 本章解决什么结构问题？

本章解决的是：

> 当问题可以被拆成阶段，并且不同决策路径会反复到达相同子问题时，如何用状态表达历史影响，并通过状态转移完成求解。

DP 的本质不是：

```text
最长
最短
方案数
```

而是：

```text
当前状态 = 若干更小状态的组合结果
```

真正重要的是：

- 状态表示什么
- 状态如何承载历史影响
- 状态之间如何转移
- 为什么这些状态已经足够
- 为什么遍历顺序不会破坏依赖关系

---

# ② DP 的统一结构视角

传统分类：

```text
一维DP
二维DP
```

只是“状态数组长什么样”。

真正重要的是：

# 状态依赖结构

因为：

```text
二维数组 ≠ 同一种DP
```

例如：

| 题目 | 本质结构 |
|---|---|
| LCS | 双序列匹配 |
| Edit Distance | 双序列匹配 |
| 区间回文 | 区间合并 |
| 二维背包 | 选择约束 |
| DAG 最长路 | 图依赖结构 |

它们虽然都用了二维数组：

```text
dp[i][j]
```

但状态依赖关系完全不同。

因此：

> 本章按“状态依赖结构”组织，而不是按数组维度组织。

---

# ③ DP 的统一建模流程

所有 DP 都遵循：

```text
1. 定义状态
2. 定义状态转移
3. 定义 base case
4. 定义遍历顺序
5. 判断是否可压缩空间
```

统一骨架：

```java
for (state in validOrder) {
    for (choice in choices) {
        dp[state] = combine(
            dp[state],
            dp[previousState]
        );
    }
}
```

核心不变量：

> 当处理当前状态时，它依赖的所有前置状态已经被正确计算。

---

# ④ 状态依赖结构分类

---

# 1）阶段推进型状态

## 对应专题

- 线性DP
- 状态机DP

## 结构定义

状态沿时间、下标或阶段自然推进。

典型形式：

```text
dp[i]
```

或：

```text
dp[i][state]
```

当前状态只依赖更早阶段。

---

## 结构信号

| 信号 | 可能结构 |
|---|---|
| 到第 i 个位置 | 线性DP |
| 前 i 个元素 | 线性DP |
| 持有 / 未持有 | 状态机DP |
| 今天和昨天有关 | 状态推进 |
| 股票交易 | 状态机DP |

---

## 代码骨架

### 线性DP

```java
for (int i = 1; i < n; i++) {
    dp[i] = transition(dp, i);
}
```

### 状态机DP

```java
for (int i = 1; i < n; i++) {
    hold = Math.max(hold, cash - prices[i]);
    cash = Math.max(cash, hold + prices[i]);
}
```

---

## 不变量

- 当前状态含义必须稳定
- 转移只能来自历史阶段
- 遍历顺序必须保证依赖已计算

---

## 容易误判

| 误判 | 实际结构 |
|---|---|
| 股票当贪心 | 状态机DP |
| 打家劫舍当回溯 | 线性DP |
| 最大子数组当滑动窗口 | 线性DP |

---

# 2）选择约束型状态

## 对应专题

- 背包DP
- 树形DP（部分）

## 结构定义

当前选择会限制未来可选集合。

核心思想：

```text
选 or 不选
```

典型状态：

```text
dp[capacity]
```

或：

```text
dp[node][selected]
```

---

## 结构信号

| 信号 | 可能结构 |
|---|---|
| 每个元素只能选一次 | 0/1 背包 |
| 元素无限使用 | 完全背包 |
| 容量限制 | 背包 |
| 选当前不能选相邻 | 选择约束 |
| 树上选点 | 树形DP |

---

## 代码骨架

### 0/1 背包

```java
for (int num : nums) {
    for (int j = target; j >= num; j--) {
        dp[j] |= dp[j - num];
    }
}
```

### 完全背包

```java
for (int coin : coins) {
    for (int j = coin; j <= amount; j++) {
        dp[j] = Math.min(dp[j], dp[j - coin] + 1);
    }
}
```

---

## 不变量

- 0/1 背包必须倒序
- 完全背包必须正序
- 状态必须足够表达历史选择影响

---

## 容易误判

| 误判 | 实际结构 |
|---|---|
| Coin Change 当 BFS | 完全背包 |
| Target Sum 当 DFS | 背包DP |
| House Robber III 当普通DFS | 树形DP |

---

# 3）双序列匹配型状态

## 对应专题

- 序列DP

## 结构定义

两个序列共同推进。

典型状态：

```text
dp[i][j]
```

表示：

```text
A 前 i 个
B 前 j 个
```

之间的子问题。

---

## 结构信号

| 信号 | 可能结构 |
|---|---|
| 两个字符串比较 | 序列DP |
| 子序列匹配 | 序列DP |
| 编辑距离 | 序列DP |
| 公共子序列 | 序列DP |

---

## 代码骨架

```java
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        if (A.charAt(i - 1) == B.charAt(j - 1)) {
            dp[i][j] = dp[i - 1][j - 1] + 1;
        } else {
            dp[i][j] = Math.max(
                dp[i - 1][j],
                dp[i][j - 1]
            );
        }
    }
}
```

---

## 不变量

- dp[i][j] 必须对应明确子问题
- 字符访问通常是 i-1 / j-1
- 空前缀初始化必须明确

---

## 容易误判

| 误判 | 实际结构 |
|---|---|
| 子串当子序列 | 序列DP |
| 编辑距离当 BFS | 序列DP |
| 回文区间当 LCS | 区间DP |

---

# 4）区间合并型状态

## 对应专题

- 区间DP

## 结构定义

小区间合并成大区间。

典型状态：

```text
dp[i][j]
```

表示：

```text
区间 [i,j]
```

的最优结果。

---

## 结构信号

| 信号 | 可能结构 |
|---|---|
| 回文区间 | 区间DP |
| 戳气球 | 区间DP |
| 合并区间 | 区间DP |
| 石子游戏 | 区间DP |

---

## 代码骨架

```java
for (int len = 2; len <= n; len++) {
    for (int i = 0; i + len - 1 < n; i++) {
        int j = i + len - 1;

        for (int k = i; k < j; k++) {
            dp[i][j] = merge(...);
        }
    }
}
```

---

## 不变量

- 小区间必须先算
- 当前区间依赖更小区间
- 切分点必须完整枚举

---

## 容易误判

| 误判 | 实际结构 |
|---|---|
| 回文子序列当双序列DP | 区间DP |
| 戳气球当贪心 | 区间DP |
| 石子游戏当 DFS | 区间DP |

---

# 5）树 / 图依赖型状态

## 对应专题

- 树形DP
- DAG DP
- 记忆化搜索

## 结构定义

状态依赖关系由树或图结构决定。

不是按下标推进。

而是：

```text
父节点依赖子节点
```

或：

```text
当前节点依赖邻居
```

---

## 结构信号

| 信号 | 可能结构 |
|---|---|
| 子树信息 | 树形DP |
| 树上路径 | 树DP |
| DAG 最长路 | 图DP |
| 重复 DFS 子问题 | 记忆化搜索 |

---

## 代码骨架

### 树形DP

```java
int[] dfs(TreeNode node) {
    int[] left = dfs(node.left);
    int[] right = dfs(node.right);

    return combine(left, right);
}
```

### 记忆化搜索

```java
int dfs(int i, int j) {
    if (memo[i][j] != 0) {
        return memo[i][j];
    }

    memo[i][j] = ...
    return memo[i][j];
}
```

---

## 不变量

- 子问题结果可缓存
- 图上 DP 必须无环
- DFS 返回值语义必须稳定

---

## 容易误判

| 误判 | 实际结构 |
|---|---|
| DAG 问题当 BFS | 图DP |
| Longest Increasing Path 当回溯 | 记忆化搜索 |
| 树最大路径和当普通 DFS | 树形DP |

---

# ⑤ 与其他结构的边界

| 容易混淆 | DP结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| DP vs 前缀和 | 状态依赖 | 区间表达 | 前缀没有决策 |
| DP vs 贪心 | 保留多状态 | 局部选择 | 贪心要求可证明 |
| DP vs 回溯 | 合并子问题 | 枚举路径 | 回溯保留路径 |
| DP vs BFS | 最优子结构 | 最短层数 | BFS 不维护状态转移 |
| 线性DP vs 状态机DP | 单状态推进 | 多状态推进 | 状态机需显式状态切换 |
| 序列DP vs 区间DP | 双序列匹配 | 区间合并 | 状态依赖方向不同 |

---

# ⑥ 本章最容易犯的错误

- dp 状态含义不稳定
- 初始化与状态定义冲突
- 遍历顺序错误
- 0/1 背包正序遍历
- 把二维数组误认为同一种DP
- 只会背公式，不会解释状态意义
- 能贪心的问题强行DP
- 把子串问题误判成子序列问题

---

# ⑦ 本章统一总结

DP 的真正核心不是：

```text
会不会写转移方程
```

而是：

```text
能否用状态正确表达历史影响
```

因此：

> DP 的本质是状态依赖结构。

真正的建模能力是：

- 知道哪些历史必须保留
- 知道哪些历史可以压缩
- 知道当前状态依赖谁
- 知道遍历顺序为什么正确

当你能从“状态依赖关系”而不是“二维数组长什么样”去理解 DP 时，才真正进入 DP 建模阶段。

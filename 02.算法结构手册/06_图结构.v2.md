# 第六章：图结构

> 核心信念：**图问题的本质不是遍历，而是在节点与边的关系中识别连通、依赖或路径优化结构。**

---

## ① 本章解决什么结构问题？

本章解决的是：

> 当元素之间存在任意关系，而不再是线性或树形层级时，如何把问题建模为节点、边和状态，并选择对应的图结构。

图结构常见目标：

| 目标 | 结构问题 |
|---|---|
| 是否能到达 | 连通性 |
| 有多少连通块 | 连通分量 |
| 是否存在依赖顺序 | 拓扑排序 |
| 从起点到终点代价最小 | 最短路径 |
| 路径上的最大边尽量小 | 瓶颈路径 |
| 全部节点连通且总代价最小 | 最小生成树 |

图题第一步不是写 DFS，而是明确：边表示什么，目标优化什么。

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| 是否连通、岛屿数量、朋友圈 | DFS / BFS / 并查集 |
| 课程表、依赖、先后顺序 | 拓扑排序 |
| 无权最短步数 | BFS |
| 正权最短路径 | Dijkstra |
| 负权边 | Bellman-Ford / SPFA 风险评估 |
| `minimize maximum edge` | 瓶颈路径 |
| 连接所有点总代价最小 | MST |
| 动态合并集合 | 并查集 |

同样是“路径”，必须区分：最少边数、最小权重和、最小最大边，三者不是同一个结构。

## ③ 本章的结构骨架

### 建图骨架

```java
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < n; i++) {
    graph.add(new ArrayList<>());
}

for (int[] edge : edges) {
    graph.get(edge[0]).add(edge[1]);
}
```

有权图：

```java
List<List<int[]>> graph = new ArrayList<>();
// int[]{neighbor, weight}
```

### 图问题建模三问

1. 节点是什么？
2. 边表示什么，是有向还是无向，有权还是无权？
3. 目标是连通、顺序、最短路径、瓶颈路径还是全局连接？

## ④ 核心模型拆解

### 1）连通分量：访问与覆盖关系

#### 结构定义

连通分量关注节点是否属于同一关系集合。它不关心路径长度，只关心能否到达。

#### 不变量

- `visited[x] == true` 表示节点 `x` 所在的可达区域已经被纳入当前遍历。
- 每次从未访问节点启动 DFS/BFS，都会发现一个新的连通分量。

#### DFS 骨架

```java
void dfs(int node, boolean[] visited, List<List<Integer>> graph) {
    visited[node] = true;

    for (int next : graph.get(node)) {
        if (!visited[next]) {
            dfs(next, visited, graph);
        }
    }
}
```

#### 并查集骨架

```java
class UnionFind {
    int[] parent;

    UnionFind(int n) {
        parent = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }

    void union(int a, int b) {
        int pa = find(a);
        int pb = find(b);
        if (pa != pb) parent[pa] = pb;
    }
}
```

#### 适用条件

- 只关心是否属于同一集合。
- 边的权重不影响目标。
- 可以离线处理合并关系。

#### 失效条件

- 需要最短路径或最小代价。
- 需要处理依赖顺序。
- 图动态删除边，并查集不易直接支持。

---

### 2）拓扑排序：有向无环依赖结构

#### 结构定义

拓扑排序处理有向依赖关系。若图是 DAG，则存在一种顺序，使每条边都从前指向后。

#### 不变量

- `indegree[x]` 表示节点 `x` 尚未满足的前置依赖数量。
- 队列中节点的入度为 0，可以安全处理。
- 如果最终处理数量小于 `n`，说明存在环。

#### Kahn 算法骨架

建图 + 入度
把入度为 0 的点入队
while 队列不空:
    弹出一个点 count++
    遍历它指向的点:
        入度--
        如果入度变 0，入队
return count == n

```java
Queue<Integer> queue = new ArrayDeque<>();
int[] indegree = new int[n];

for (int i = 0; i < n; i++) {
    if (indegree[i] == 0) {
        queue.offer(i);
    }
}

int seen = 0;
while (!queue.isEmpty()) {
    int node = queue.poll();
    seen++;

    for (int next : graph.get(node)) {
        indegree[next]--;
        if (indegree[next] == 0) {
            queue.offer(next);
        }
    }
}

boolean acyclic = seen == n;
```

#### 适用条件

- 边表示先后依赖。
- 图有向。
- 需要判断是否可完成或输出一种合法顺序。

#### 失效条件

- 无向图没有拓扑意义。
- 有环时不存在完整拓扑序。
- 目标是代价最小路径，不是依赖顺序。

---

### 3）BFS 最短步数：无权图的层级扩展

#### 结构定义

无权图中，每条边代价相同。BFS 按层扩展，第一次到达某节点时就是最短步数。

#### 不变量

- 队列中节点按距离非递减顺序出队。
- `dist[x]` 第一次设置时即为最短距离。
- 每一层扩展代表路径长度增加 1。

#### 代码骨架

```java
Queue<Integer> queue = new ArrayDeque<>();
int[] dist = new int[n];
Arrays.fill(dist, -1);

queue.offer(start);
dist[start] = 0;

while (!queue.isEmpty()) {
    int node = queue.poll();

    for (int next : graph.get(node)) {
        if (dist[next] == -1) {
            dist[next] = dist[node] + 1;
            queue.offer(next);
        }
    }
}
```

#### 适用条件

- 边权相同或没有边权。
- 求最少步数、最少操作次数、最短层数。

#### 失效条件

- 边权不同，应转向 Dijkstra 或其他最短路。
- 状态包含额外维度，需要把 `(node, state)` 作为新节点。

---

### 4）Dijkstra：正权图中的最短路径

#### 结构定义

Dijkstra 维护从起点到每个节点的当前最小距离，并使用堆优先处理距离最小的候选节点。

#### 不变量

- `dist[x]` 是当前已知最短距离上界。
- 堆中保存待确认的候选状态。
- 当状态以最小距离出堆且未过期时，该距离已经稳定。

#### 代码骨架

```java
PriorityQueue<int[]> pq =
    new PriorityQueue<>((a, b) -> Integer.compare(a[1], b[1]));

int[] dist = new int[n];
Arrays.fill(dist, Integer.MAX_VALUE);

dist[start] = 0;
pq.offer(new int[]{start, 0});

while (!pq.isEmpty()) {
    int[] curr = pq.poll();
    int node = curr[0];
    int d = curr[1];

    if (d > dist[node]) continue;

    for (int[] edge : graph.get(node)) {
        int next = edge[0];
        int weight = edge[1];
        if (dist[next] > d + weight) {
            dist[next] = d + weight;
            pq.offer(new int[]{next, dist[next]});
        }
    }
}
```

#### 适用条件

- 边权非负。
- 目标是路径权重和最小。
- 需要从一个源点扩展最短距离。

#### 失效条件

- 存在负权边。
- 优化目标不是权重和，而是路径最大边、概率乘积或其他聚合方式。
- 图规模太大时需要估算复杂度和状态数量。

---

### 5）瓶颈路径：优化路径上的最大值

#### 结构定义

瓶颈路径优化的不是边权和，而是路径上的最大边最小：

```text
cost(path) = max(edgeWeight)
```

#### 不变量

- `dist[x]` 表示到达 `x` 的路径中，当前已知最小瓶颈值。
- 经过一条边后的新瓶颈是 `max(current, weight)`。

#### 代码骨架

```java
int[] dist = new int[n];
Arrays.fill(dist, Integer.MAX_VALUE);
dist[start] = 0;

PriorityQueue<int[]> pq =
    new PriorityQueue<>((a, b) -> Integer.compare(a[1], b[1]));
pq.offer(new int[]{start, 0});

while (!pq.isEmpty()) {
    int[] curr = pq.poll();
    int node = curr[0];
    int cost = curr[1];

    if (cost > dist[node]) continue;

    for (int[] edge : graph.get(node)) {
        int next = edge[0];
        int weight = edge[1];
        int nextCost = Math.max(cost, weight);
        if (nextCost < dist[next]) {
            dist[next] = nextCost;
            pq.offer(new int[]{next, nextCost});
        }
    }
}
```

#### 适用条件

- 目标形如最小化路径最大代价。
- 可以用 Dijkstra 变形、二分 + BFS，或按边排序并查集。

#### 失效条件

- 目标是总和最小，直接用标准最短路。
- 需要全图连接最优，可能是 MST。

---

### 6）最小生成树：全局连通代价最小

#### 结构定义

MST 要在无向连通图中选择若干边，使所有节点连通且总权重最小。它不是求两点之间最短路径。

#### Kruskal 骨架

```java
Arrays.sort(edges, (a, b) -> Integer.compare(a[2], b[2]));

UnionFind uf = new UnionFind(n);
int total = 0;
int used = 0;

for (int[] edge : edges) {
    int a = edge[0];
    int b = edge[1];
    int w = edge[2];

    if (uf.find(a) != uf.find(b)) {
        uf.union(a, b);
        total += w;
        used++;
    }
}
```

#### 适用条件

- 无向图。
- 目标是连接所有节点。
- 优化总边权，而不是某两点路径。

#### 失效条件

- 有向依赖问题。
- 起点到终点路径最短问题。
- 图不连通，需要先判断可连接性。

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| 图 DFS vs 树 DFS | 可能有环和多路径 | 天然层级无环 | 图需要访问状态和环判断 |
| BFS vs Dijkstra | 无权最短步数 | 正权最短路径 | 边权不同不能用普通 BFS |
| 最短路径 vs 瓶颈路径 | 权重和最小 | 最大边最小 | 路径代价聚合方式不同 |
| MST vs 最短路径 | 全局连通最优 | 单源或两点路径最优 | MST 不保证两点最短 |
| 拓扑排序 vs BFS | 依赖入度消除 | 层级扩展 | 拓扑处理有向依赖 |

---

## ⑥ 本章最容易误判的地方

- 没有明确图是有向还是无向。
- 把无权 BFS 用到有权图。
- 用 Dijkstra 处理负权边。
- 把瓶颈路径误判为最短路径。
- 把 MST 当作两点最短路。
- 忘记二维网格也可以建模为图。
- 状态压缩图中只记录节点，不记录附加状态。

---

## ⑦ 小结

图结构的统一视角是：

> 先抽象节点和边，再识别目标是连通、依赖、路径代价还是全局连接。

图题的难点通常不在遍历代码，而在建模和目标函数。目标函数一变，结构就可能从 BFS 变成 Dijkstra、从最短路变成瓶颈路径，或从路径问题变成生成树问题。

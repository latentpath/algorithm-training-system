# 第十一章：设计类数据结构

> 核心信念：**设计题的本质不是实现功能，而是在长期动态状态中组合结构以满足操作复杂度。**

---

## ① 本章解决什么结构问题？

本章解决的是：

> 当一个对象需要长期支持多种操作，并且每种操作都有复杂度要求时，如何拆解状态、选择结构并保持多结构同步。

普通算法题通常是：

```text
输入 -> 计算 -> 输出
```

设计类数据结构是：

```text
初始化 -> 多次操作 -> 持续维护不变量
```

核心不是某一次计算，而是每次操作后结构仍然正确。

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| `get / put` 要求 `O(1)` | HashMap + 辅助结构 |
| 最近使用 / 最近淘汰 | HashMap + 双向链表 |
| 频率优先，再按时间淘汰 | 多 Map + 频率链表 |
| 前缀查询、自动补全 | Trie |
| 随机返回且插入删除 `O(1)` | ArrayList + HashMap |
| 数据流中位数 | 双堆 |
| 栈中查询最小值 | 主栈 + 辅助栈 |

设计题的入口通常是复杂度约束，而不是题目名字。

---

## ③ 本章的结构骨架

设计题建模流程：

```text
1. 列出所有操作
2. 标注每个操作的复杂度要求
3. 明确需要维护的状态
4. 判断单一结构能否满足所有操作
5. 设计多个结构之间的同步不变量
```

关键问题：

- 查询靠什么结构快？
- 更新靠什么结构快？
- 顺序或优先级靠什么结构维护？
- 删除时哪些结构必须同时更新？

---

## ④ 核心模型拆解

### 1）HashMap：随机访问与定位结构

#### 结构定义

HashMap 在设计题中通常负责从 key 快速定位到值、节点或索引。

#### 不变量

- `map` 中的 key 与主结构中的元素一一对应。
- 主结构移动或删除元素时，`map` 必须同步更新。

#### 常见映射

```text
key -> value
key -> Node
value -> index
freq -> list of nodes
```

#### 适用条件

- 需要按 key `O(1)` 查询。
- 需要快速定位某个节点或数组位置。

#### 失效条件

- 需要有序查询、范围查询，应考虑 TreeMap。
- 需要按优先级取最值，应使用堆或有序结构。

---

### 2）双向链表：O(1) 维护顺序关系

#### 结构定义

双向链表维护节点之间的时间顺序、频率组内顺序或插入顺序。它支持在已知节点引用时 `O(1)` 删除和移动。

#### 不变量

- 头尾哨兵简化边界。
- 每个真实节点必须同时被前驱和后继连接。
- `map` 指向的节点必须仍在链表中。

#### 节点骨架

```java
class Node {
    int key;
    int value;
    Node prev;
    Node next;
}
```

#### 操作骨架

```java
void remove(Node node) {
    node.prev.next = node.next;
    node.next.prev = node.prev;
}

void addFirst(Node node) {
    node.next = head.next;
    node.prev = head;
    head.next.prev = node;
    head.next = node;
}
```

#### 适用条件

- 已经能通过 Map 定位节点。
- 需要 `O(1)` 移动、删除或维护时间顺序。

#### 失效条件

- 需要按值有序查询。
- 只需要栈或队列单端操作，双向链表可能过重。

---

### 3）LRU：随机访问 + 时间顺序淘汰

#### 结构定义

LRU 维护“最近使用”顺序：

```text
访问或更新 -> 移动到最近端
容量满 -> 淘汰最久未使用端
```

#### 结构组合

| 结构 | 作用 |
|---|---|
| HashMap | `key -> Node`，O(1) 定位 |
| DoubleList | O(1) 维护最近使用顺序 |

#### 不变量

- Map 中所有节点都在链表中。
- 链表头部表示最近使用，尾部表示最久未使用。
- 每次 `get` 或命中 `put` 都更新顺序。

#### 骨架

```java
class LRUCache {
    Map<Integer, Node> map = new HashMap<>();
    DoubleList list = new DoubleList();
    int capacity;

    int get(int key) {
        if (!map.containsKey(key)) return -1;
        Node node = map.get(key);
        list.remove(node);
        list.addFirst(node);
        return node.value;
    }

    void put(int key, int value) {
        if (map.containsKey(key)) {
            Node node = map.get(key);
            node.value = value;
            list.remove(node);
            list.addFirst(node);
            return;
        }

        if (map.size() == capacity) {
            Node old = list.removeLast();
            map.remove(old.key);
        }

        Node node = new Node(key, value);
        list.addFirst(node);
        map.put(key, node);
    }
}
```

#### 失效条件

- 淘汰规则不是时间顺序，而是频率或多维优先级，应转向 LFU。

---

### 4）LFU：频率与时间的分层优先级

#### 结构定义

LFU 同时维护两个优先级：

```text
频率越低越先淘汰
频率相同时，越久未使用越先淘汰
```

#### 结构组合

| 状态 | 含义 |
|---|---|
| `key -> value` | 数据 |
| `key -> freq` | 当前频率 |
| `freq -> LinkedHashSet/key list` | 同频率下的时间顺序 |
| `minFreq` | 当前最小频率 |

#### 不变量

- 每个 key 只存在于一个频率组。
- 访问 key 后必须从旧频率组移动到新频率组。
- 旧的 `minFreq` 组为空时，`minFreq` 必须更新。

#### 适用条件

- 淘汰规则由频率和时间共同决定。
- 操作要求接近 `O(1)`。

#### 失效条件

- 只需要时间顺序，LRU 更简单。
- 需要复杂权重排序，可能需要 TreeMap 或堆。

---

### 5）Trie：公共前缀共享结构

#### 结构定义

Trie 用路径共享公共前缀。它不是普通树遍历题，而是字符串集合的前缀索引结构。

#### 不变量

- 从根到某节点的路径表示一个前缀。
- `isEnd` 区分“前缀存在”和“完整单词存在”。

#### 代码骨架

```java
class TrieNode {
    TrieNode[] children = new TrieNode[26];
    boolean isEnd;
}

void insert(String word) {
    TrieNode node = root;

    for (char c : word.toCharArray()) {
        int index = c - 'a';
        if (node.children[index] == null) {
            node.children[index] = new TrieNode();
        }
        node = node.children[index];
    }

    node.isEnd = true;
}
```

#### 适用条件

- 大量字符串共享前缀。
- 需要前缀查询、自动补全或词典匹配。

#### 失效条件

- 只需要精确字符串查询，HashSet 更简单。
- 字符集很大或稀疏时，应改用 `Map<Character, TrieNode>`。

### 6）RandomizedSet：动态数组 + 索引同步

#### 结构定义

RandomizedSet 的核心矛盾是：

```text
O(1) 删除任意值
O(1) 随机访问
```

#### 结构组合

| 结构 | 作用 |
|---|---|
| ArrayList | O(1) 随机访问 |
| HashMap | value -> index |

#### 不变量

- `map.get(value)` 必须是 value 在数组中的真实下标。
- 删除中间元素时，用最后一个元素覆盖删除位置，并同步更新索引。

#### 代码骨架

```java
List<Integer> nums = new ArrayList<>();
Map<Integer, Integer> index = new HashMap<>();

void remove(int val) {
    int i = index.get(val);
    int last = nums.get(nums.size() - 1);

    nums.set(i, last);
    index.put(last, i);

    nums.remove(nums.size() - 1);
    index.remove(val);
}
```

#### 失效条件

- 需要保持插入顺序。
- 随机概率不是均匀分布。

### 7）辅助结构同步：MinStack 与 MedianFinder

#### MinStack

MinStack 通过辅助栈同步维护当前最小值。

```java
Stack<Integer> stack = new Stack<>();
Stack<Integer> minStack = new Stack<>();

void push(int val) {
    stack.push(val);
    if (minStack.isEmpty()) {
        minStack.push(val);
    } else {
        minStack.push(Math.min(val, minStack.peek()));
    }
}
```

不变量：`minStack.peek()` 始终对应当前主栈的最小值。

#### MedianFinder

MedianFinder 用双堆维护动态平衡。

```java
PriorityQueue<Integer> small =
    new PriorityQueue<>((a, b) -> Integer.compare(b, a));
PriorityQueue<Integer> large = new PriorityQueue<>();
```

不变量：

- `small` 保存较小一半。
- `large` 保存较大一半。
- 两堆大小差不超过 1。

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| 设计题 vs 普通算法题 | 长期维护状态 | 一次性计算答案 | 每次操作后都要保持不变量 |
| Trie vs 树递归 | 前缀索引 | 层级信息汇总 | Trie 的边表示字符 |
| LRU vs 堆 | 时间顺序淘汰 | 当前优先级最值 | LRU 需要 O(1) 移动节点 |
| RandomizedSet vs 数组 | 数组负责随机访问 | Map 负责定位 | 删除必须同步索引 |
| MedianFinder vs 排序 | 动态平衡 | 一次性有序 | 数据流不能每次重排 |

---

## ⑥ 本章最容易误判的地方

- 只考虑查询复杂度，忽略更新复杂度。
- 多结构之间状态不同步。
- 删除节点时忘记同步 Map。
- LRU 命中 `get` 后没有更新最近使用顺序。
- LFU 更新频率后忘记维护 `minFreq`。
- Trie 中混淆“前缀存在”和“单词结束”。
- RandomizedSet 删除最后一个元素时索引更新顺序错误。

---

## ⑦ 小结

设计类数据结构的统一视角是：

> 把复杂操作拆成查询、更新、顺序和同步四类需求，再用多个基础结构共同维护不变量。

真正重要的不是用了哪个容器，而是每次操作后所有结构之间是否仍然一致。

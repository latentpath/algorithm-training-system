# 前言

这本手册不是算法题解合集。

它的核心目标是：

> 帮助你在 10~30 秒内识别题目的“结构本质”。

多数人依赖“题型记忆”，而本手册强调的是：

- 结构识别
- 目标函数识别
- 误判排除
- 边界区分

当你不再问“这题像不像某题”，而是问“这题属于哪种结构？”时，算法能力就进入了体系化阶段。

# 使用指南

## 一、做题流程

1. 先识别结构（而不是直接想算法）
2. 确认目标函数（min / max / count / 是否存在 / 所有方案等）
3. 排除相似结构（参考第13章 结构对照总表）
4. 再落地具体算法和模板

## 二、结构识别优先级顺序

看到题目后，依次思考：

1. 是否存在图关系或连通性？
2. 是否是状态转移 / 最优决策问题？
3. 是否需要枚举所有可能（回溯）？
4. 是否存在单调性或极值问题？
5. 是否是区间 / 连续子数组问题？
6. 是否需要动态维护极值（单调栈/队列）？
7. 是否是线性指针移动？
8. 是否是二分判定？

## 三、如何高效复盘

每道题复盘时写三行：

- 结构类型：
- 目标函数：
- 为什么不是其他结构：

能清楚说出“为什么不是”，才说明真正理解了该结构。

# 第九章：回溯与搜索结构

> 核心信念：**回溯不是暴力枚举，而是在决策树上维护路径、不变量和剪枝边界。**

---

## ① 本章解决什么结构问题？

本章解决的是：

> 当答案来自一组选择序列，且需要枚举所有可行解或在搜索空间中寻找可行解时，如何系统展开决策树并控制无效分支。

回溯结构的核心抽象：

```text
节点 = 一个部分选择状态
边 = 做出一个新选择
路径 = 当前已经做出的选择序列
剪枝 = 当前路径不可能通向合法答案
```

它处理的是“选择空间”，不是数组或图的天然结构。

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| 列出所有子集 / 组合 / 排列 | 回溯枚举 |
| 所有可行切分、所有路径 | 决策树搜索 |
| N 皇后、数独 | 约束搜索 |
| 是否存在一种方案 | DFS + 剪枝 / 记忆化 |
| 搜索空间指数级但规模较小 | 回溯 |
| 重复子问题明显 | 记忆化搜索 / DP |

回溯成立的关键是能画出选择树，并定义每一层的可选集合。

---

## ③ 本章的结构骨架

回溯三要素：

```text
选择
递归
撤销选择
```

通用骨架：

```java
void backtrack(State state, List<Integer> path) {
    if (isAnswer(state)) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (Choice choice : choices(state)) {
        if (!valid(choice, state)) continue;

        apply(choice, state, path);
        backtrack(state, path);
        undo(choice, state, path);
    }
}
```

不变量：

> 递归进入下一层前，`path` 与辅助状态准确表示当前部分解；递归返回后必须恢复到进入前的状态。

## ④ 核心模型拆解

### 1）子集结构：每个元素选或不选

#### 结构定义

子集结构枚举所有可能的元素集合。常用递归参数 `start` 保证后续选择只从右侧继续，避免重复。

#### 不变量

- `path` 是当前已经选择的元素集合。
- 下一层只能从 `start` 之后选择。
- 每个子集只生成一次。

#### 代码骨架

```java
void dfs(int start, List<Integer> path) {
    result.add(new ArrayList<>(path));

    for (int i = start; i < nums.length; i++) {
        path.add(nums[i]);
        dfs(i + 1, path);
        path.remove(path.size() - 1);
    }
}
```

#### 适用条件

- 需要枚举所有子集。
- 元素相对顺序不重要，但不能重复使用。

#### 失效条件

- 只需要子集数量或最优值，DP 或数学计数可能更合适。
- 元素有重复且需要去重，必须排序并控制同层重复。

---

### 2）组合结构：固定数量或固定目标

#### 结构定义

组合结构从候选集中选若干元素，顺序不重要。

#### 不变量

- `start` 控制每个元素最多使用一次。
- `path.size()` 或当前和控制终止。
- 同层选择不能重复产生同一组合。

#### 代码骨架

```java
void dfs(int start, int k, List<Integer> path) {
    if (path.size() == k) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int i = start; i <= n; i++) {
        path.add(i);
        dfs(i + 1, k, path);
        path.remove(path.size() - 1);
    }
}
```

#### 剪枝边界

```java
int need = k - path.size();
for (int i = start; i <= n - need + 1; i++) {
    ...
}
```

#### 适用条件

- 顺序不重要。
- 需要枚举固定大小或满足目标的选择集合。

#### 失效条件

- 顺序重要，应转向排列结构。
- 存在大量重复状态，应考虑 DP。

---

### 3）排列结构：每一层选择一个未使用元素

#### 结构定义

排列结构按位置逐步填充答案，每一层选择一个尚未使用的元素。

#### 不变量

- `path` 长度表示已经填充的位置数。
- `used[i]` 表示元素 `i` 是否已在当前路径中。
- 每个元素在一条路径中最多使用一次。

#### 代码骨架

```java
void dfs(List<Integer> path, boolean[] used) {
    if (path.size() == nums.length) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int i = 0; i < nums.length; i++) {
        if (used[i]) continue;

        used[i] = true;
        path.add(nums[i]);

        dfs(path, used);

        path.remove(path.size() - 1);
        used[i] = false;
    }
}
```

#### 适用条件

- 顺序不同代表不同答案。
- 需要填充每个位置。

#### 失效条件

- 只关心集合，不关心顺序，应使用组合 / 子集。
- 需要最短交换或最少操作，可能是 BFS 或图搜索。

---

### 4）切分结构：在位置之间做决策

#### 结构定义

切分结构在字符串或数组的边界之间选择切分点。每一层决定下一段 `[start, end]`。

#### 不变量

- `start` 表示下一段的起点。
- `path` 中每一段都已经满足局部合法性。
- 当 `start == n` 时，完整切分形成答案。

#### 代码骨架

```java
void dfs(int start, List<String> path) {
    if (start == s.length()) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int end = start; end < s.length(); end++) {
        if (!valid(s, start, end)) continue;

        path.add(s.substring(start, end + 1));
        dfs(end + 1, path);
        path.remove(path.size() - 1);
    }
}
```

#### 适用条件

- 答案由若干连续片段组成。
- 每段合法性可独立判断。

#### 失效条件

- 只需要最少切分次数，DP 更合适。
- 合法性判断昂贵，应预处理或记忆化。

---

### 5）剪枝与记忆化：控制搜索空间

#### 剪枝结构

剪枝不是随意 `return`，而是基于“不可能形成合法答案”的证明。

常见剪枝：

```java
if (currentSum > target) return;
if (remaining < need) return;
if (conflict(row, col)) continue;
```

#### 去重边界

排序后，同一层跳过重复选择：

```java
if (i > start && nums[i] == nums[i - 1]) {
    continue;
}
```

#### 记忆化搜索

当不同路径到达同一状态时，回溯会重复计算。此时应把状态缓存：

```java
Map<State, Boolean> memo = new HashMap<>();
```

#### 适用条件

- 剪枝条件能被结构证明。
- 状态可被稳定编码。

#### 失效条件

- 路径本身影响答案输出，不能简单缓存状态。
- 状态空间已经适合自底向上 DP。

---

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| 回溯 vs DFS 图 | 人为决策树 | 图的节点边关系 | 图搜索需要 visited 处理环 |
| 回溯 vs DP | 枚举路径 | 合并重复状态 | DP 不保留具体选择路径 |
| 回溯 vs 贪心 | 尝试多种选择 | 每步选择可证明最优 | 贪心不回退 |
| 回溯 vs BFS | 深度优先枚举 | 层级最短扩展 | 最少步数常用 BFS |
| 回溯 vs 位运算 | 搜索结构 | 状态编码优化 | bitmask 常用来加速 used 状态 |

---

## ⑥ 本章最容易误判的地方

- 忘记撤销选择，导致兄弟分支状态污染。
- `start` 与 `used` 混用，组合和排列边界混乱。
- 去重条件放错层级。
- 剪枝条件过强，剪掉合法答案。
- 把子数组连续结构误写成组合搜索。
- 存在大量重复子问题却不使用记忆化或 DP。

---

## ⑦ 小结

回溯结构的统一视角是：

> 在决策树中维护当前路径，用合法性约束控制分支，用撤销操作恢复递归不变量。

回溯前先画出搜索树：每一层代表什么，每个分支选择什么，什么时候形成答案，什么时候可以停止。搜索树清楚，代码才会稳定。

# 第五章：树与递归信息结构

> 核心信念：**树问题的本质不是遍历，而是在层级结构中组织信息流。**

---

## ① 本章解决什么结构问题？

本章解决的是：

> 当问题天然具有父子层级关系时，如何定义递归函数，使信息在节点之间正确传递、汇总和返回。

数组结构强调边界移动，树结构强调递归信息：

```text
父 -> 子：传递路径、约束或上下文
子 -> 父：返回高度、合法性、答案候选
左右子树 -> 当前节点：合并局部结果
```

树题的核心问题通常不是“怎么遍历”，而是：

> `dfs(node)` 到底表示什么，返回什么，当前节点如何使用子树信息。

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| `subtree`、高度、直径、平衡 | 子树信息汇总 |
| `path sum`、根到叶路径 | 路径状态递归 |
| `lowest common ancestor` | 分治返回 |
| `BST`、第 k 小、范围校验 | 有序递归结构 |
| `level order`、最小深度 | 层序 BFS |
| 偷家劫舍 III、树上选择 | 树形 DP |
| 序列化 / 反序列化 | 结构编码 |

判断入口不是遍历顺序，而是信息流方向。

---

## ③ 本章的结构骨架

### 递归信息骨架

```java
Info dfs(TreeNode node) {
    if (node == null) {
        return baseCase;
    }

    Info left = dfs(node.left);
    Info right = dfs(node.right);

    Info current = merge(node, left, right);
    return current;
}
```

### 树题建模三问

1. `dfs(node)` 代表以 `node` 为根的什么问题？
2. 空节点返回什么，为什么？
3. 当前节点如何利用左、右子树信息得到自己的返回值？

---

## ④ 核心模型拆解

### 1）遍历型结构：访问顺序本身有意义

#### 结构定义

遍历型结构关注节点访问顺序，而不是复杂信息汇总。

常见顺序：

```text
前序：根 -> 左 -> 右
中序：左 -> 根 -> 右
后序：左 -> 右 -> 根
```

#### 不变量

- 前序适合父节点先提供上下文。
- 中序在 BST 中产生有序序列。
- 后序适合先拿到子树信息再处理当前节点。

#### 代码骨架

```java
void inorder(TreeNode node) {
    if (node == null) return;

    inorder(node.left);
    visit(node);
    inorder(node.right);
}
```

#### 适用条件

- 问题结果由访问顺序直接产生。
- BST 有序性可通过中序表达。
- 不需要复杂返回信息。

#### 失效条件

- 需要从子树返回多个状态。
- 需要维护路径并撤销状态。
- 图结构存在环，需要 `visited`。

---

### 2）子树信息汇总：后序递归返回状态

#### 结构定义

子树汇总让 `dfs(node)` 返回以 `node` 为根的子树信息，例如高度、是否合法、最大贡献。

#### 不变量

- `left` 是左子树完整信息。
- `right` 是右子树完整信息。
- 当前节点只基于 `node`、`left`、`right` 计算返回值。

#### 高度骨架

```java
int depth(TreeNode node) {
    if (node == null) return 0;

    int left = depth(node.left);
    int right = depth(node.right);

    return Math.max(left, right) + 1;
}
```

#### 平衡树骨架

```java
int dfs(TreeNode node) {
    if (node == null) return 0;

    int left = dfs(node.left);
    if (left == -1) return -1;

    int right = dfs(node.right);
    if (right == -1) return -1;

    if (Math.abs(left - right) > 1) return -1;
    return Math.max(left, right) + 1;
}
```

#### 适用条件

- 答案依赖子树信息。
- 子树之间只在父节点处合并。
- 局部返回值能代表整个子树。

#### 失效条件

- 需要跨子树维护全局路径历史。
- 子问题有重复且不是树结构，应考虑图或 DP。
- 只需要层级最短距离时，BFS 更直接。

---

### 3）路径型结构：根到当前节点的状态维护

#### 结构定义

路径型结构维护递归栈上的一条路径：

```text
root -> ... -> node
```

它更接近树上的回溯。

#### 不变量

- 进入节点前，`path` 表示父节点路径。
- 加入当前节点后，`path` 表示根到当前节点路径。
- 离开节点前必须撤销当前节点，恢复父层状态。

#### 代码骨架

```java
void dfs(TreeNode node, List<Integer> path) {
    if (node == null) return;

    path.add(node.val);

    if (node.left == null && node.right == null) {
        record(path);
    }

    dfs(node.left, path);
    dfs(node.right, path);

    path.remove(path.size() - 1);
}
```

#### 适用条件

- 目标是根到叶、祖先到当前、路径和等问题。
- 状态随递归深入而扩展，回退时需要恢复。

#### 失效条件

- 题目要求任意两点路径，往往需要子树贡献或 LCA。
- 路径状态可重复计算并有重叠子问题时，可考虑记忆化。

---

### 4）分治返回结构：左右结果在当前节点合并

#### 结构定义

分治结构中，左右子树分别返回结果，当前节点判断是否形成答案。

最近公共祖先是典型模型：如果左右子树分别找到目标，当前节点就是汇合点。

#### 代码骨架

```java
TreeNode lca(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) {
        return root;
    }

    TreeNode left = lca(root.left, p, q);
    TreeNode right = lca(root.right, p, q);

    if (left != null && right != null) {
        return root;
    }

    return left != null ? left : right;
}
```

#### 适用条件

- 当前节点可以根据左右子树返回值决定答案。
- 结果天然按子树分解。

#### 失效条件

- 图中存在多条路径或环。
- 需要全局最短路径，BFS 或图算法更合适。

---

### 5）BST：有序性进入递归结构

#### 结构定义

BST 不是普通二叉树。它把有序约束嵌入递归结构：

```text
左子树 < 当前节点 < 右子树
```

并且每棵子树仍是 BST。

#### 不变量

- 中序遍历产生升序序列。
- 搜索路径具有类似二分的唯一方向。
- 校验时每个节点必须满足来自祖先的范围约束。

#### 校验骨架

```java
boolean validate(TreeNode node, long low, long high) {
    if (node == null) return true;

    if (node.val <= low || node.val >= high) {
        return false;
    }

    return validate(node.left, low, node.val)
        && validate(node.right, node.val, high);
}
```

#### 适用条件

- 题目明确是 BST，或需要利用中序有序性。
- 操作依赖有序搜索、范围约束或第 k 小。

#### 失效条件

- 只是普通二叉树，不能使用 BST 性质。
- 存在重复值时，必须先明确重复值放置规则。

---

### 6）树形 DP：子树状态转移

#### 结构定义

树形 DP 的状态不是下标，而是节点：

```text
dp(node) = 以 node 为根的子树状态
```

一个节点常需要返回多个状态，以表达“选 / 不选”“覆盖 / 不覆盖”等关系。

#### 代码骨架

```java
int[] dfs(TreeNode node) {
    if (node == null) {
        return new int[]{0, 0};
    }

    int[] left = dfs(node.left);
    int[] right = dfs(node.right);

    int choose = node.val + left[1] + right[1];
    int skip = Math.max(left[0], left[1])
             + Math.max(right[0], right[1]);

    return new int[]{choose, skip};
}
```

#### 适用条件

- 当前节点状态依赖左右子树状态。
- 同一子树状态可抽象为有限维返回值。
- 存在局部选择与子树约束。

#### 失效条件

- 状态不只由子树决定，还依赖复杂祖先历史。
- 图结构有环，需要图 DP 或记忆化搜索。

---

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| 树 DFS vs 图 DFS | 无环层级信息流 | 关系连通与访问控制 | 图需要 `visited` 和环处理 |
| 树路径 vs 回溯 | 固定树结构上的路径状态 | 决策树枚举 | 回溯的分支是人为选择 |
| 树形 DP vs 线性 DP | 子树状态 | 下标阶段状态 | 遍历顺序由树结构决定 |
| BST vs 二分 | 递归有序结构 | 连续判定空间 | BST 的边界来自祖先范围 |
| 层序 BFS vs DFS | 按层扩展 | 按递归深度处理 | 最短层数通常 BFS 更稳 |

---

## ⑥ 本章最容易误判的地方

- 不清楚 `dfs(node)` 返回什么。
- 把所有树题都当遍历题，忽略信息汇总。
- 前序、后序信息流混用。
- 空节点 base case 与返回语义不一致。
- BST 校验只比较父子节点，忽略祖先范围。
- 路径题忘记回溯恢复状态。
- Tree DP 返回状态维度不够。

---

## ⑦ 小结

树结构的统一视角是：

> 在递归层级中定义信息的流向、含义和合并方式。

写树题前先定义 `dfs(node)`。只要返回值、base case 和合并逻辑清楚，遍历顺序通常会自然出现。

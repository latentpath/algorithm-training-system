# 第三章：区间结构

> 核心信念：**区间问题的本质是排序后扫描，并维护当前区间关系或全局覆盖状态。**

---

## ① 本章解决什么结构问题？

本章解决多个区间之间的关系建模问题：

- 是否重叠。
- 如何合并。
- 如何插入并维持有序结构。
- 如何选择最多或最优的不重叠区间。
- 如何统计某一时刻的覆盖数量。

区间结构不是“看到 interval 就合并”。它首先要判断：

> 当前问题是在整理区间关系、做区间选择，还是统计事件变化？

---

## ② 本章的结构判断入口

### 结构识别信号

| 信号 | 可能结构 |
|---|---|
| `start / end`、重叠、合并 | 区间合并 |
| 插入一个新区间并保持不重叠 | 区间插入 |
| 选择最多不重叠区间 | 区间调度 |
| 删除最少区间使剩余不重叠 | 区间选择 / 贪心 |
| 最大同时覆盖数、会议室数量 | 扫描线 |
| 大量区间加减、离线统计 | 差分 / 扫描线 |

区间问题的第一步通常是排序，但排序维度取决于结构目标。

---

## ③ 本章的结构骨架

通用流程：

```text
1. 明确区间边界语义：闭区间、半开区间还是时间点事件
2. 选择排序维度：start、end 或事件点
3. 线性扫描
4. 维护当前区间、已选集合或 active 状态
```

常见不变量：

```text
current        当前正在维护的合并区间
result         已经确定且不再被后续区间影响的区间
active         当前时间点仍在覆盖中的区间数量
end            已选择区间集合的最后结束位置
```

---

## ④ 核心模型拆解

### 1）区间合并：维护当前覆盖段

#### 结构定义

区间合并将有重叠关系的区间整理为互不重叠的覆盖段。

排序后，扫描到新区间 `next` 时只需要判断它是否与 `current` 重叠：

```text
next.start <= current.end
```

#### 不变量

- `result` 中的区间已经确定，后续区间不会再与它们重叠。
- `current` 是当前扫描位置前仍可能继续扩张的覆盖段。
- 未扫描区间按 `start` 不递减排列。

#### 代码骨架

```java
Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

List<int[]> result = new ArrayList<>();
int[] current = intervals[0];

for (int i = 1; i < intervals.length; i++) {
    int[] next = intervals[i];
    if (next[0] <= current[1]) {
        current[1] = Math.max(current[1], next[1]);
    } else {
        result.add(current);
        current = next;
    }
}

result.add(current);
```

#### 适用条件

- 目标是整理覆盖范围，而不是优化选择。
- 区间可按起点排序。
- 重叠关系可以由当前合并段和下一个区间判断。

#### 失效条件

- 目标是选择最多区间，不能合并。
- 目标是统计同时存在的数量，应转向扫描线。
- 区间有复杂权重，需要 DP 或贪心证明。

---

### 2）区间插入：三段式扫描

#### 结构定义

在已有不重叠且按起点排序的区间列表中插入新区间。

扫描可以分成三段：

```text
完全在新区间左侧
与新区间重叠，需要合并
完全在新区间右侧
```

#### 代码骨架

```java
List<int[]> result = new ArrayList<>();
int i = 0;

while (i < n && intervals[i][1] < newInterval[0]) {
    result.add(intervals[i++]);
}

while (i < n && intervals[i][0] <= newInterval[1]) {
    newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
    newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
    i++;
}

result.add(newInterval);

while (i < n) {
    result.add(intervals[i++]);
}
```

#### 适用条件

- 原区间已经有序且互不重叠。
- 插入后仍需保持整理后的区间结构。

#### 失效条件

- 原始区间无序或本身重叠，需要先合并。
- 插入是动态高频操作，需要平衡树或有序映射。

---

### 3）区间选择：按结束时间维护可扩展空间

#### 结构定义

区间选择从多个区间中选择尽量多的不重叠区间。

核心不是合并，而是保留后续空间。排序维度应是结束时间。

#### 不变量

- 已选择区间互不重叠。
- `end` 是已选择集合的最后结束位置。
- 在同等选择数量下，结束越早，留给未来的空间越大。

#### 代码骨架

```java
Arrays.sort(intervals, (a, b) -> Integer.compare(a[1], b[1]));

int count = 0;
int end = Integer.MIN_VALUE;

for (int[] interval : intervals) {
    if (interval[0] >= end) {
        count++;
        end = interval[1];
    }
}
```

#### 适用条件

- 目标是最大化不重叠区间数量，或最小化删除数量。
- 区间权重相同。
- “结束更早更优”的交换论证成立。

#### 失效条件

- 区间有权重，选择数量不是唯一目标，应转向 DP。
- 重叠规则不是简单的时间边界。
- 需要维护动态加入删除的区间集合。

---

### 4）扫描线：把区间转为事件变化

#### 结构定义

扫描线不直接处理区间本身，而是处理区间端点导致的状态变化。

```text
start: active + 1
end:   active - 1
```

它解决的是某个位置上的全局覆盖状态，例如最大重叠数。

#### 不变量

- 事件按时间顺序处理。
- `active` 表示当前时间点仍在覆盖中的区间数。
- 同一时间点的开始和结束顺序必须符合区间边界语义。

#### 代码骨架

```java
List<int[]> events = new ArrayList<>();

for (int[] interval : intervals) {
    events.add(new int[]{interval[0], 1});
    events.add(new int[]{interval[1], -1});
}

events.sort((a, b) -> {
    if (a[0] != b[0]) return Integer.compare(a[0], b[0]);
    return Integer.compare(a[1], b[1]);
});

int active = 0;
int answer = 0;

for (int[] event : events) {
    active += event[1];
    answer = Math.max(answer, active);
}
```

#### 适用条件

- 目标是统计覆盖数量、峰值或状态变化。
- 问题可离线处理全部事件。
- 区间端点可以排序。

#### 失效条件

- 需要在线响应查询，可能需要堆、平衡树或线段树。
- 区间权重和事件规则复杂，需要维护更多状态。
- 只需整理覆盖段时，合并比扫描线更直接。

## ⑤ 与其他章节的边界

| 容易混淆 | 本章结构 | 另一个结构 | 核心区别 |
|---|---|---|---|
| 合并 vs 选择 | 整理覆盖关系 | 优化保留集合 | 合并会丢失原区间个体，选择保留个体 |
| 扫描线 vs 前缀 | 离散事件累积 | 下标累积表达 | 扫描线是事件变化的前缀 |
| 区间 vs 指针 | 排序后的二维边界关系 | 原序线性边界移动 | 区间先排序，指针通常保留原顺序 |
| 扫描线 vs 堆 | 离线统计事件 | 在线维护当前最早结束 | 会议室可两者建模，在线场景堆更自然 |
| 区间选择 vs DP | 无权贪心选择 | 有权状态转移 | 有权区间调度通常需要 DP |

---

## ⑥ 本章最容易误判的地方

- 把区间选择误写成区间合并。
- 合并时忘记加入最后一个 `current`。
- 排序维度与结构目标不一致。
- 忽略闭区间、开区间、半开区间的端点顺序。
- 扫描线同一点 start / end 的处理顺序错误。
- 有权区间仍套用无权贪心。

---

## ⑦ 小结

区间结构的统一视角是：

> 先明确区间关系，再选择排序维度，最后在线性扫描中维护状态。

当目标是整理覆盖，用合并；当目标是保留最多个体，用结束时间选择；当目标是统计全局变化，用扫描线。结构目标不同，排序维度和不变量也不同。

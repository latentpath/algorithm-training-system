# 二分查找

## 本章目标

训练“单调性”的第一反应：看到有序、答案范围、最小可行值、最大可行值，快速套二分。

## 秒判卡

| 看到什么 | 直接想到 |
|---|---|
| 有序数组查找 | 普通二分 |
| 第一个大于等于 | lower bound |
| 最后一个小于等于 | upper bound 变体 |
| 最小满足条件 | first true |
| 最大满足条件 | last true |
| 答案在一个范围内 | 二分答案 |
| `minimize maximum` | 二分答案 |
| `maximize minimum` | 二分答案 |

## 高频模板 1：找第一个满足条件

### 秒判关键词

第一个、左边界、lower bound、最小可行。

### 母题

LC704 / lower bound：找第一个 `>= target` 的位置。

### 标准代码

```java
public int lowerBound(int[] nums, int target) {
    int left = 0, right = nums.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] >= target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;
}
```

### 一句话记忆

满足条件就收右边，最后 `left` 是第一个 true。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Search Insert Position | 返回 lower bound |
| Find First and Last Position | 左边界 + 右边界 |
| First Bad Version | `isBad(mid)` 是条件 |

### 常见坑点

- `right = nums.length` 是左闭右开写法。
- 返回后要检查是否越界、是否等于目标。

## 高频模板 2：找最后一个满足条件

### 秒判关键词

最后一个、右边界、最大可行。

### 母题

找最后一个 `<= target` 的位置。

### 标准代码

```java
public int upperBoundLast(int[] nums, int target) {
    int left = 0, right = nums.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] <= target) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }
    return left - 1;
}
```

### 一句话记忆

先找第一个大于目标，再往左退一格。

### 高频变体

- 目标出现区间：`first >= target` 到 `first > target - 1`。
- 最大可行答案：条件成立时向右试。

### 常见坑点

- `left - 1` 可能是 `-1`。
- 不要混用闭区间和左闭右开更新规则。

## 高频模板 3：二分答案

### 秒判关键词

最小最大值、最大最小值、答案范围、可行性检查。

### 母题

LC875 Koko Eating Bananas：最小吃香蕉速度。

### 标准代码

```java
public int minEatingSpeed(int[] piles, int h) {
    int left = 1, right = 0;
    for (int p : piles) right = Math.max(right, p);

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (canFinish(piles, h, mid)) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;
}

private boolean canFinish(int[] piles, int h, int speed) {
    long hours = 0;
    for (int p : piles) {
        hours += (p + speed - 1) / speed;
    }
    return hours <= h;
}
```

### 一句话记忆

二分答案 = 猜答案 + check 是否可行。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Capacity To Ship Packages | check 能否在 D 天运完 |
| Split Array Largest Sum | check 分段数是否不超过 k |
| Magnetic Force Between Balls | check 是否能放下 m 个球 |

### 常见坑点

- 先确定答案上下界。
- `check(mid)` 必须有单调性。
- 累加可能溢出，用 `long`。

## 高频模板 4：旋转数组二分

### 秒判关键词

旋转有序数组、局部有序、找目标或最小值。

### 母题

LC153 Find Minimum in Rotated Sorted Array

### 标准代码

```java
public int findMin(int[] nums) {
    int left = 0, right = nums.length - 1;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] > nums[right]) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }
    return nums[left];
}
```

### 一句话记忆

和右端比，`mid` 大说明最小值在右边。

### 高频变体

- Search in Rotated Sorted Array：判断哪半边有序。
- 有重复元素：相等时 `right--`，最坏退化到 `O(n)`。

### 常见坑点

- 不要直接和 `nums[0]` 混乱比较。
- 有重复元素时单调判断会变弱。

## 高频坑点

| 坑点 | 正确反应 |
|---|---|
| 死循环 | 用统一模板，不混边界 |
| `mid = (l+r)/2` 溢出 | `l + (r-l)/2` |
| 二分答案不会写 check | 先写暴力可行性 |
| 返回位置没检查 | 越界或值不等都可能 |
| 条件没有单调性 | 不能二分 |

## 一题多变体

```text
Search Insert Position
↓
First Bad Version
↓
Find First and Last Position
```

```text
Koko Eating Bananas
↓

↓

```

## 面试速记

- 二分不是只看有序数组，关键是单调性。
- 左边界：满足就 `right = mid`。
- 最大可行：可行就向右试。
- 二分答案：先定范围，再写 `check`。
- 不确定边界时优先左闭右开。

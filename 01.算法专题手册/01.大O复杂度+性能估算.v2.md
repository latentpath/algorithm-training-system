# 大O复杂度 + 性能估算

## 本章目标

训练面试中的第一反应：看到 `n`、循环、递归、数据规模，快速判断这份代码或方案能不能过。

## 秒判卡

| 看到什么 | 直接想到 |
|---|---|
| `n <= 10` | 指数 / 回溯可能能过 |
| `n <= 20` | `2^n` 勉强考虑 |
| `n <= 10^3` | `O(n^2)` 常见可过 |
| `n <= 10^5` | `O(n log n)` / `O(n)` |
| `n <= 10^9` | `O(log n)` / 数学 |
| 两层遍历数组 | 先估 `O(n^2)` |
| 排序后扫描 | `O(n log n)` |
| 每步减半 | `O(log n)` |
| 子集 / 所有组合 | `O(2^n)` |
| 全排列 | `O(n!)` |

## 高频模板 1：按数据规模反推算法

### 秒判关键词

`n` 很大、问能不能过、选择复杂度。

### 母题

面试中给定 `n = 10^5`，判断双重循环是否可用。

### 标准判断

```text
1 秒大约看作 10^8 次简单操作。

n = 10^5:
- O(n) 可以
- O(n log n) 可以
- O(n^2) 基本不行
```

### 一句话记忆

`10^5` 看到平方直接警觉，优先想排序、哈希、双指针、堆。

### 高频变体

| 数据规模 | 常见可接受复杂度 |
|---|---|
| `10` | `O(n!)` / `O(2^n)` |
| `20` | `O(2^n)` |
| `100` | `O(n^3)` |
| `1000` | `O(n^2)` |
| `10^5` | `O(n log n)` |
| `10^9` | `O(log n)` |

### 常见坑点

- `10^5 * 10^5` 不是“慢一点”，是基本不可用。
- `O(n log n)` 通常来自排序、堆、分治。
- `O(log n)` 通常来自二分、堆高度、快速幂。

## 高频模板 2：循环复杂度

### 秒判关键词

循环、嵌套循环、指针只前进。

### 母题

判断两段循环和双指针扫描的复杂度。

### 标准判断

```java
// O(n)
for (int i = 0; i < n; i++) {}

// O(n^2)
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {}
}

// O(n)，不是 O(n^2)
int left = 0, right = 0;
while (right < n) {
    right++;
    while (left < right && needShrink()) {
        left++;
    }
}
```

### 一句话记忆

看总移动次数，不看 while 嵌套外形。

### 高频变体

- 滑动窗口：`left`、`right` 都最多走 `n` 次，所以 `O(n)`。
- 双指针夹逼：左右指针总共移动 `n` 次，所以 `O(n)`。
- 每次从头重新找：通常变成 `O(n^2)`。

### 常见坑点

- 看到嵌套 `while` 不一定是 `O(n^2)`。
- 看到两个 `for` 也不一定能优化，关键看指针是否回退。

## 高频模板 3：递归复杂度速判

### 秒判关键词

递归、分治、回溯、树形调用。

### 母题

判断二分查找、归并排序、子集枚举。

### 标准判断

| 递归形式 | 第一反应 |
|---|---|
| `T(n)=T(n/2)+O(1)` | `O(log n)` |
| `T(n)=T(n/2)+O(n)` | `O(n)` |
| `T(n)=2T(n/2)+O(n)` | `O(n log n)` |
| 每个元素选 / 不选 | `O(2^n)` |
| 每层选择一个未用元素 | `O(n!)` |

### 一句话记忆

分治看“分几份”，回溯看“每层多少选择”。

### 高频变体

- 二分查找：每次只进一边，`O(log n)`。
- 归并排序：两边都做，还要合并，`O(n log n)`。
- 子集：每个元素两个选择，`O(2^n)`。
- 排列：第一层 `n` 种，第二层 `n-1` 种，`O(n!)`。

### 常见坑点

- 二叉树 DFS 是 `O(n)`，因为每个节点访问一次。
- 回溯不是看代码行数，是看搜索树大小。

## 高频模板 4：空间复杂度

### 秒判关键词

额外数组、哈希表、递归栈、原地修改。

### 母题

判断反转数组、DFS、HashMap 计数。

### 标准判断

| 做法 | 空间 |
|---|---|
| 原地交换数组 | `O(1)` |
| 新建等长数组 | `O(n)` |
| HashMap 计数 | `O(n)` |
| 二叉树递归 DFS | `O(h)` |
| 回溯路径数组 | `O(n)`，结果集另算 |

### 一句话记忆

空间看额外开了什么，递归也算栈。

### 高频变体

- 平衡树递归深度：`O(log n)`。
- 链状树递归深度：`O(n)`。
- 返回所有答案时，结果集通常不算进辅助空间，但面试要说明。

### 常见坑点

- `Arrays.sort` 具体空间依语言实现，面试中先说排序主成本是时间。
- 递归不是 `O(1)` 空间。

## 高频坑点

| 坑点 | 正确反应 |
|---|---|
| `n=10^5` 写双重循环 | 优先找 `O(n log n)` 或 `O(n)` |
| 嵌套 while 误判平方 | 看指针是否只前进 |
| 递归忘算栈 | DFS 空间通常是高度 |
| 回溯只写 `O(n)` | 通常是指数级 |
| 字符串循环拼接 | Java 用 `StringBuilder` |

## 一题多变体

```text
数据规模判断
↓
循环总移动次数
↓
递归搜索树大小
↓
空间是否有额外结构
```

## 面试速记

- `10^5`：别写平方。
- 排序：`O(n log n)`。
- 哈希：常见 `O(n)` 时间、`O(n)` 空间。
- 双指针 / 滑动窗口：指针不回退就是 `O(n)`。
- 回溯：子集 `2^n`，排列 `n!`。
- 递归空间：看最大调用深度。

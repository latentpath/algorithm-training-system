# 贪心算法 Greedy

## 本章目标

训练高频贪心题的第一反应：看到区间、跳跃、股票、调度，先想局部最优是否能稳定推进。

## 秒判卡

| 看到什么 | 直接想到 |
|---|---|
| 区间不重叠最多 | 按结束时间排序 |
| 最少删除重叠区间 | 保留结束最早的 |
| 合并区间 | 按起点排序 |
| 跳跃到终点 | 维护最远可达 |
| 最少跳几次 | 当前层最远边界 |
| 股票多次交易 | 累加所有上涨段 |
| 分发饼干 / 匹配 | 排序 + 双指针 |

## 高频模板 1：区间调度

### 秒判关键词

最多不重叠、最少移除、会议安排。

### 母题

LC435 Non-overlapping Intervals

### 标准代码

```java
public int eraseOverlapIntervals(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> Integer.compare(a[1], b[1]));
    int keep = 0;
    int end = Integer.MIN_VALUE;

    for (int[] in : intervals) {
        if (in[0] >= end) {
            keep++;
            end = in[1];
        }
    }
    return intervals.length - keep;
}
```

### 一句话记忆

想保留最多区间，就优先选结束最早的。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Minimum Number of Arrows | 重叠区间共用一箭 |
| Meeting Rooms | 判断是否有重叠 |
| Meeting Rooms II | 扫描线或最小堆 |

### 常见坑点

- 区间调度排序按结束时间，不是开始时间。
- 合并区间才按开始时间排序。

## 高频模板 2：跳跃游戏

### 秒判关键词

能否到达、最少跳跃、最远可达。

### 母题

LC55 Jump Game

### 标准代码

```java
public boolean canJump(int[] nums) {
    int farthest = 0;
    for (int i = 0; i < nums.length; i++) {
        if (i > farthest) return false;
        farthest = Math.max(farthest, i + nums[i]);
    }
    return true;
}
```

### 一句话记忆

一路维护最远可达，当前位置超过最远就失败。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Jump Game II | 当前跳跃范围结束时步数 +1 |
| Gas Station | 总油量够，再找起点 |
| Video Stitching | 类似覆盖区间的最远右端 |

### 常见坑点

- Jump Game II 是层次贪心，别每一步跳当前最大。
- 到达最后一个位置即可，不一定要跳出数组。

## 高频模板 3：股票多次交易

### 秒判关键词

可以买卖多次、最大利润、每次只持有一股。

### 母题

LC122 Best Time to Buy and Sell Stock II

### 标准代码

```java
public int maxProfit(int[] prices) {
    int profit = 0;
    for (int i = 1; i < prices.length; i++) {
        if (prices[i] > prices[i - 1]) {
            profit += prices[i] - prices[i - 1];
        }
    }
    return profit;
}
```

### 一句话记忆

多次交易时，把所有上涨段都吃掉。

### 高频变体

- Best Time I：只能一次，维护最低买入价。
- With Fee / Cooldown：通常改 DP，不硬贪心。

### 常见坑点

- 只能交易一次不能累加所有上涨。
- 有手续费或冷冻期时优先 DP。

## 高频模板 4：排序 + 双指针匹配

### 秒判关键词

最小满足、分配资源、匹配胃口和饼干。

### 母题

LC455 Assign Cookies

### 标准代码

```java
public int findContentChildren(int[] g, int[] s) {
    Arrays.sort(g);
    Arrays.sort(s);
    int i = 0, j = 0;

    while (i < g.length && j < s.length) {
        if (s[j] >= g[i]) {
            i++;
        }
        j++;
    }
    return i;
}
```

### 一句话记忆

用能满足当前需求的最小资源去匹配。

### 高频变体

- Boats to Save People：最重的人优先，能带最轻就带。
- Task Scheduler：计数贪心 / 堆。

### 常见坑点

- 贪心前通常要排序。
- 匹配题要明确“先满足谁”。

## 高频坑点

| 坑点 | 正确反应 |
|---|---|
| 所有题都想贪心 | 先找局部选择能否不后悔 |
| 区间排序维度错 | 调度按结束，合并按开始 |
| 股票有限制还贪心 | 手续费 / 冷冻期转 DP |
| Jump Game II 逐步选 | 维护当前层边界和下一层最远 |
| 匹配没排序 | 先排序再双指针 |

## 一题多变体


## 面试速记

- 区间调度：结束越早越好。
- 合并区间：起点排序。
- 跳跃：维护最远可达。
- 多次股票：吃上涨段。
- 贪心题要能说清“为什么当前选择不会吃亏”。

# 堆 / 优先队列 / Top K

## 本章目标

训练“动态最值”的第一反应：看到 Top K、第 k 大、实时中位数、调度最早结束，快速想到堆。

## 秒判卡

| 看到什么 | 直接想到 |
|---|---|
| 第 k 大 / 前 k 大 | 小顶堆维护 k 个 |
| 第 k 小 / 前 k 小 | 大顶堆维护 k 个 |
| 数据流中位数 | 双堆 |
| 每次取当前最小 / 最大 | PriorityQueue |
| 合并 k 个有序链表 | 小顶堆 |
| 会议室 / 调度最早结束 | 小顶堆 |
| 固定窗口最大值 | 单调队列优先，不是堆 |

## 高频模板 1：静态 Top K

### 秒判关键词

第 k 大、前 k 高频、只关心 k 个最好。

### 母题

LC215 Kth Largest Element in an Array

### 标准代码

```java
public int findKthLargest(int[] nums, int k) {
    PriorityQueue<Integer> pq = new PriorityQueue<>();
    for (int x : nums) {
        pq.offer(x);
        if (pq.size() > k) {
            pq.poll();
        }
    }
    return pq.peek();
}
```

### 一句话记忆

求第 k 大，用小顶堆保留最大的 k 个。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Top K Frequent Elements | 先 Map 计数，再堆按频次 |
| K Closest Points | 堆按距离 |
| Last Stone Weight | 大顶堆，每次取最大两个 |

### 常见坑点

- 第 k 大用小顶堆，不是大顶堆。
- 堆大小超过 k 就弹出最弱的。

## 高频模板 2：合并 k 个有序结构

### 秒判关键词

k 个有序链表、多个有序数组、每次取最小头部。

### 母题

LC23 Merge k Sorted Lists

### 标准代码

```java
public ListNode mergeKLists(ListNode[] lists) {
    PriorityQueue<ListNode> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a.val));
    for (ListNode node : lists) {
        if (node != null) pq.offer(node);
    }

    ListNode dummy = new ListNode(0);
    ListNode cur = dummy;
    while (!pq.isEmpty()) {
        ListNode node = pq.poll();
        cur.next = node;
        cur = cur.next;
        if (node.next != null) pq.offer(node.next);
    }
    return dummy.next;
}
```

### 一句话记忆

堆里永远放每条链当前最小的候选。

### 高频变体

- Kth Smallest in Sorted Matrix：堆放每行当前元素。
- Smallest Range Covering K Lists：堆 + 当前最大值。

### 常见坑点

- 空链表不要入堆。
- 弹出一个节点后，把它的下一个节点入堆。

## 高频模板 3：数据流中位数

### 秒判关键词

不断加入数字、随时查询中位数。

### 母题

LC295 Find Median from Data Stream

### 标准代码

```java
class MedianFinder {
    PriorityQueue<Integer> small = new PriorityQueue<>(Collections.reverseOrder());
    PriorityQueue<Integer> large = new PriorityQueue<>();

    public void addNum(int num) {
        if (small.isEmpty() || num <= small.peek()) {
            small.offer(num);
        } else {
            large.offer(num);
        }

        if (small.size() > large.size() + 1) {
            large.offer(small.poll());
        } else if (large.size() > small.size()) {
            small.offer(large.poll());
        }
    }

    public double findMedian() {
        if (small.size() > large.size()) return small.peek();
        return (small.peek() + large.peek()) / 2.0;
    }
}
```

### 一句话记忆

大顶堆放较小一半，小顶堆放较大一半。

### 高频变体

- Sliding Window Median：双堆 + 延迟删除。
- IPO：一个堆按资本，一个堆按利润。

### 常见坑点

- 两个堆大小差不能超过 1。
- 中位数求平均要用 `/ 2.0`。

## 高频模板 4：调度类优先队列

### 秒判关键词

会议室、任务调度、最早结束、释放资源。

### 母题

LC253 Meeting Rooms II

### 标准代码

```java
public int minMeetingRooms(int[][] intervals) {
    Arrays.sort(intervals, Comparator.comparingInt(a -> a[0]));
    PriorityQueue<Integer> ends = new PriorityQueue<>();

    for (int[] in : intervals) {
        if (!ends.isEmpty() && ends.peek() <= in[0]) {
            ends.poll();
        }
        ends.offer(in[1]);
    }
    return ends.size();
}
```

### 一句话记忆

小顶堆维护当前占用资源的最早释放时间。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Meeting Rooms II | 堆中是结束时间 |
| Single-Threaded CPU | 按时间推进，堆按处理时长 |
| Task Scheduler | 计数 + 冷却，可用堆或公式 |

### 常见坑点

- 先按开始时间排序。
- 一个会议开始前，结束时间 `<= start` 的房间可复用。

## 高频坑点

| 坑点 | 正确反应 |
|---|---|
| 第 k 大用错堆 | 小顶堆保留 k 个最大 |
| 比较器溢出 | 用 `Integer.compare` |
| 数据流中位数堆不平衡 | 每次 add 后 rebalance |
| 固定窗口最大值用堆复杂处理 | 单调队列更直接 |
| 调度忘排序 | 先按开始时间 |

## 面试速记

- Top K：堆大小固定为 k。
- 第 k 大：小顶堆。
- 第 k 小：大顶堆。
- 多路合并：堆放每路当前头。
- 中位数：双堆平衡。
- 调度：小顶堆维护最早结束。

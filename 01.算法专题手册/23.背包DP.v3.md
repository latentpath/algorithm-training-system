# 背包 DP

## 本章目标

训练“选或不选”的第一反应：看到容量、目标和、组合数、最值，优先想到背包 DP。

---

## 秒判卡

| 看到什么 | 直接想到 |
|---|---|
| 每个元素只能用一次 | 0/1 背包 |
| 元素可以无限使用 | 完全背包 |
| 求方案数 | 组合 / 排列计数 |
| 求最大值 / 最小值 | max / min DP |
| 能否凑出来 | boolean DP |
| 恰好装满 | 初始化特殊 |

---

## 高频模板 1：0/1 背包

### 秒判关键词

每个元素只能选一次、是否能凑出、子集和。

### 母题

LC416 Partition Equal Subset Sum

### 标准代码

```java
public boolean canPartition(int[] nums) {
    int sum = 0;
    for (int x : nums) sum += x;

    if ((sum & 1) == 1) return false;

    int target = sum / 2;
    boolean[] dp = new boolean[target + 1];
    dp[0] = true;

    for (int num : nums) {
        for (int j = target; j >= num; j--) {
            dp[j] = dp[j] || dp[j - num];
        }
    }

    return dp[target];
}
```

### 一句话记忆

每个数只能用一次，所以容量必须倒序。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Last Stone Weight II | 转成最接近一半 |
| Target Sum | 转成子集和计数 |
| Ones and Zeroes | 二维容量 |

### 常见坑点

- 0/1 背包必须倒序。
- `dp[j]` 表示容量 j 是否可达。
- target 往往来自总和的一半或变形公式。

## 高频模板 2：完全背包最小值

### 秒判关键词

硬币无限使用、最少数量、凑金额。

### 母题

LC322 Coin Change

### 标准代码

```java
public int coinChange(int[] coins, int amount) {
    int INF = amount + 1;
    int[] dp = new int[amount + 1];

    Arrays.fill(dp, INF);
    dp[0] = 0;

    for (int coin : coins) {
        for (int j = coin; j <= amount; j++) {
            dp[j] = Math.min(dp[j], dp[j - coin] + 1);
        }
    }

    return dp[amount] == INF ? -1 : dp[amount];
}
```

### 一句话记忆

元素可重复，所以容量正序。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Perfect Squares | 硬币变成平方数 |
| Word Break | 位置 DP + 字典判断 |
| Minimum Cost For Tickets | 选择不同通行证 |

### 常见坑点

- 最小值初始化为大数，不是 0。
- 完全背包容量正序遍历。
- 返回前要判断是否仍是 INF。

## 高频模板 3：组合数背包

### 秒判关键词

有多少种组合，不关心顺序。

### 母题

LC518 Coin Change II

### 标准代码

```java
public int change(int amount, int[] coins) {
    int[] dp = new int[amount + 1];
    dp[0] = 1;

    for (int coin : coins) {
        for (int j = coin; j <= amount; j++) {
            dp[j] += dp[j - coin];
        }
    }

    return dp[amount];
}
```

### 一句话记忆

组合数：先物品，再容量。

### 常见坑点

- `dp[0] = 1` 表示凑出 0 有一种空方案。
- 先容量会把不同顺序算成不同方案。

## 高频模板 4：排列数背包

### 秒判关键词

不同顺序算不同方案。

### 母题

LC377 Combination Sum IV

### 标准代码

```java
public int combinationSum4(int[] nums, int target) {
    int[] dp = new int[target + 1];
    dp[0] = 1;

    for (int sum = 1; sum <= target; sum++) {
        for (int x : nums) {
            if (sum >= x) {
                dp[sum] += dp[sum - x];
            }
        }
    }

    return dp[target];
}
```

### 一句话记忆

排列数：先容量，再物品。

### 常见坑点

- 组合和排列只差循环顺序，但语义完全不同。
- 注意题目是否说“order matters”。

## 高频模板 5：Target Sum 转子集和

### 母题

LC494 Target Sum

### 标准代码

```java
public int findTargetSumWays(int[] nums, int target) {
    int sum = 0;
    for (int x : nums) sum += x;

    if (Math.abs(target) > sum) return 0;
    if (((sum + target) & 1) == 1) return 0;

    int pos = (sum + target) / 2;
    int[] dp = new int[pos + 1];
    dp[0] = 1;

    for (int num : nums) {
        for (int j = pos; j >= num; j--) {
            dp[j] += dp[j - num];
        }
    }

    return dp[pos];
}
```

### 一句话记忆

正数集合 P，负数集合 N：`P - N = target`，`P + N = sum`。

---

## 高频遍历顺序总结

| 类型 | 遍历 |
|---|---|
| 0/1 背包 | 容量倒序 |
| 完全背包 | 容量正序 |
| 组合数 | 先物品，后容量 |
| 排列数 | 先容量，后物品 |

---

## 面试速记

- 先判断：能否重复使用。
- 再判断：求最值、可行性还是方案数。
- 最后判断：组合还是排列。
- 遍历顺序决定题目类型。

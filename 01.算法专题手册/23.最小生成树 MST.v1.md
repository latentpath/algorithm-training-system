# 最小生成树 MST

## 本章目标

训练“全局连通代价最小”的第一反应：看到连接所有点、修路组网、最低总成本，快速想到 MST。

## 秒判卡

| 看到什么 | 直接想到 |
|---|---|
| 连接所有城市最低成本 | MST |
| 铺设网络总费用最小 | MST |
| 保证所有点连通 | MST |
| 选择 n - 1 条边 | MST |
| 无向带权图全连通 | MST |
| 两点最短距离 | 最短路径，不是 MST |
| 有向依赖顺序 | 拓扑排序，不是 MST |

## MST vs 最短路径

### MST 解决什么

```text
用最小总代价，
把所有节点连成一个整体。
```

关注的是：

```text
全局连接成本
```

---

### 最短路径解决什么

```text
从起点到终点，
路径代价最小。
```

关注的是：

```text
局部路径最优
```

---

### 本质区别

| 问题 | 算法 |
|---|---|
| 全部节点连通总代价最小 | MST |
| 单点到单点距离最小 | 最短路径 |
| 单源到所有点最短路 | Dijkstra |
| 无权最短步数 | BFS |

---

### 一句话判断

```text
连所有点 → MST

求两点距离 → 最短路
```

## 高频模板 1：Kruskal 最小生成树

### 秒判关键词

连接所有点、最低成本、无向带权图。

### 母题

LC1584 Min Cost to Connect All Points

### 核心思想

```text
按边权从小到大排序。

如果一条边连接了两个原本不连通的集合，
就选它。
```

本质：

```text
贪心 + 并查集
```

---

### 标准代码

```java
class UnionFind {
    int[] parent;
    int[] size;

    UnionFind(int n) {
        parent = new int[n];
        size = new int[n];

        for (int i = 0; i < n; i++) {
            parent[i] = i;
            size[i] = 1;
        }
    }

    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }

    boolean union(int a, int b) {
        int ra = find(a);
        int rb = find(b);

        if (ra == rb) return false;

        if (size[ra] < size[rb]) {
            parent[ra] = rb;
            size[rb] += size[ra];
        } else {
            parent[rb] = ra;
            size[ra] += size[rb];
        }

        return true;
    }
}

public int kruskal(int n, int[][] edges) {
    Arrays.sort(edges, (a, b) -> Integer.compare(a[2], b[2]));

    UnionFind uf = new UnionFind(n);

    int total = 0;
    int used = 0;

    for (int[] e : edges) {
        int a = e[0];
        int b = e[1];
        int w = e[2];

        if (uf.union(a, b)) {
            total += w;
            used++;

            if (used == n - 1) break;
        }
    }

    return total;
}
```

### 一句话记忆

从小边开始选，只要不会成环就加入。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Min Cost to Connect All Points | 曼哈顿距离建边 |
| Connecting Cities With Minimum Cost | 城市建图 |
| Optimize Water Distribution | 虚拟源点 |
| Critical and Pseudo-Critical Edges | MST 关键边 |

### 常见坑点

- MST 是无向图问题。
- `union` 成功才加入边权。
- MST 最终应使用 `n - 1` 条边。
- 图不连通时无法形成 MST。

## 高频模板 2：Prim 最小生成树

### 秒判关键词

图稠密、从点扩展、邻接表。

### 母题

Prim 标准模板

### 核心思想

```text
每次从：
“当前连通块”
向外扩展一条最小边。
```

它更像：

```text
Dijkstra 的“连通版”
```

---

### 标准代码

```java
public int prim(int n, List<int[]>[] graph) {
    boolean[] visited = new boolean[n];

    PriorityQueue<int[]> pq =
        new PriorityQueue<>(Comparator.comparingInt(a -> a[1]));

    pq.offer(new int[]{0, 0});

    int total = 0;
    int used = 0;

    while (!pq.isEmpty()) {
        int[] cur = pq.poll();

        int node = cur[0];
        int weight = cur[1];

        if (visited[node]) continue;

        visited[node] = true;
        total += weight;
        used++;

        for (int[] next : graph[node]) {
            int nei = next[0];
            int w = next[1];

            if (!visited[nei]) {
                pq.offer(new int[]{nei, w});
            }
        }
    }

    return used == n ? total : -1;
}
```

### 一句话记忆

不断用最小边，把新点拉进当前连通块。

### 高频变体

- 稠密图 Prim 通常更自然。
- 有些题显式给邻接矩阵。

### 常见坑点

- Prim 不是最短路径。
- 堆里存的是“边”，不是“距离”。
- visited 后不能重复加入总代价。

## 高频模板 3：关键边 / 伪关键边

### 秒判关键词

关键边、必须边、可替代边。

### 母题

LC1489 Find Critical and Pseudo-Critical Edges

---

### 核心思想

#### 关键边

```text
删掉它后：
MST 变大
或者无法连通
```

说明：

```text
它必须出现
```

---

#### 伪关键边

```text
强制加入它，
仍然能形成最优 MST
```

说明：

```text
它可以出现
但不是必须
```

---

### 标准思路

```text
先求原始 MST 权值。

对于每条边：

1. 禁用该边，再跑 Kruskal
   如果答案变差：
   它是关键边

2. 强制使用该边，再跑 Kruskal
   如果答案不变：
   它是伪关键边
```

### 一句话记忆

删掉变差的是关键边，强行加入还能最优的是伪关键边。

### 高频变体

- 边分类问题。
- MST 唯一性判断。

### 常见坑点

- 边排序后要保留原始编号。
- “无法连通”也算关键边。

## 高频模板 4：虚拟源点 MST

### 秒判关键词

每个点可单独建设、井、水源、电站。

### 母题

LC1168 Optimize Water Distribution in a Village

---

### 核心思想

如果：

```text
每个点自己建设
也算一种连接方式
```

那么：

```text
增加一个虚拟节点 0
```

例如：

```text
0 -> i
边权 = 单独建设成本
```

然后：

```text
统一跑 MST
```

---

### 一句话记忆

“单独建设”转成“虚拟源点连边”。

### 高频变体

- 供水系统
- 电网建设
- 基站覆盖

### 常见坑点

- 虚拟点也参与 MST。
- 总节点数会变成 `n + 1`。

## MST vs 并查集

### 并查集解决什么

```text
动态连通性
```

例如：

- 是否连通
- 是否成环
- 合并集合

---

### MST 为什么常用并查集

Kruskal 需要：

```text
快速判断：
加这条边会不会成环
```

因此：

```text
用并查集维护连通块
```

---

### 一句话记忆

并查集是工具，MST 是目标。

## Kruskal vs Prim

| 算法 | 核心 |
|---|---|
| Kruskal | 从边扩展 |
| Prim | 从点扩展 |

---

### Kruskal 特点

```text
排序所有边
+
并查集判环
```

适合：

- 稀疏图
- 边容易枚举

---

### Prim 特点

```text
从当前连通块
不断扩展最小边
```

适合：

- 稠密图
- 邻接表天然存在

---

### 面试建议

```text
LeetCode 高频：
Kruskal 更重要
```

## 高频坑点

| 坑点 | 正确反应 |
|---|---|
| MST 当最短路径 | 两者目标不同 |
| 有向图跑 MST | MST 通常是无向图 |
| union 原节点 | union 根节点 |
| 边没排序 | Kruskal 必须排序 |
| used 不等于 n - 1 | 图不连通 |
| Prim 重复加点 | visited 后跳过 |

## 一题多变体

```text
Min Cost to Connect All Points
↓
Connecting Cities With Minimum Cost
↓
Optimize Water Distribution
```

```text
标准 MST
↓

```


## 结构判断总表

| 目标 | 结构 |
|---|---|
| 两点最短距离 | 最短路径 |
| 无权最短步数 | BFS |
| 全局连通总代价最小 | MST |
| 判断是否连通 | 并查集 |
| 有向依赖顺序 | 拓扑排序 |

## 面试速记

- MST：连接所有点总代价最小。
- Kruskal：排序边 + 并查集。
- Prim：从点扩展最小边。
- MST 不是最短路径。
- 最终边数应为 `n - 1`。

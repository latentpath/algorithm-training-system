# 哈希表

## 本章目标

训练“快速查找”和“频次统计”的第一反应：看到配对、去重、计数、出现过，优先想到 HashMap / HashSet。

## 秒判卡

| 看到什么 | 直接想到 |
|---|---|
| 是否出现过 | HashSet |
| 计数 / 频率 | HashMap |
| 两数之和 | HashMap 查 complement |
| 字母异位词 | 计数表 |
| 子数组和为 K | 前缀和 + HashMap |
| 最长无重复子串 | 滑动窗口 + HashSet/Map |
| O(n²) 查找太慢 | 用哈希降一维 |

## 高频模板 1：两数之和

### 秒判关键词

配对、目标和、找另一个数。

### 母题

LC1 Two Sum

### 标准代码

```java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int need = target - nums[i];
        if (map.containsKey(need)) {
            return new int[]{map.get(need), i};
        }
        map.put(nums[i], i);
    }
    return new int[0];
}
```

### 一句话记忆

遍历当前数，去表里找它缺的另一半。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Contains Duplicate | Set 判断是否出现过 |
| 3Sum | 排序 + 双指针，不直接三重哈希 |
| 4Sum II | 两边分组，把 `A+B` 计数 |

### 常见坑点

- 先查再放，避免同一个元素用两次。
- HashMap 存下标还是次数，要看题目要返回什么。

## 高频模板 2：频次计数

### 秒判关键词

字符频率、异位词、出现次数相同。

### 母题

LC242 Valid Anagram

### 标准代码

```java
public boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) return false;

    int[] count = new int[26];
    for (int i = 0; i < s.length(); i++) {
        count[s.charAt(i) - 'a']++;
        count[t.charAt(i) - 'a']--;
    }
    for (int c : count) {
        if (c != 0) return false;
    }
    return true;
}
```

### 一句话记忆

小写字母用数组，通用元素用 HashMap。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Group Anagrams | 排序字符串或 26 维计数作 key |
| Find All Anagrams | 滑动窗口维护计数差 |
| First Unique Character | 先计数，再扫描 |

### 常见坑点

- 字符范围不确定时不要硬写 `26`。
- Map 计数减到 0 可以删除，方便比较窗口状态。

## 高频模板 3：前缀和 + 哈希表

### 秒判关键词

连续子数组、和为 K、求个数。

### 母题

LC560 Subarray Sum Equals K

### 标准代码

```java
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> count = new HashMap<>();
    count.put(0, 1);

    int sum = 0;
    int ans = 0;
    for (int x : nums) {
        sum += x;
        ans += count.getOrDefault(sum - k, 0);
        count.put(sum, count.getOrDefault(sum, 0) + 1);
    }
    return ans;
}
```

### 一句话记忆

当前前缀和是 `sum`，找之前有没有 `sum - k`。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Continuous Subarray Sum | 前缀和取模 |
| Longest Subarray Sum K | Map 存最早下标 |
| Subarray Sums Divisible by K | 统计余数出现次数 |

### 常见坑点

- 初始化 `count.put(0, 1)`。
- 先查再放，避免空子数组干扰。
- 有负数时不能用普通滑动窗口求和。

## 高频模板 4：滑动窗口哈希

### 秒判关键词

子串、无重复、包含字符、窗口内频次。

### 母题

LC3 Longest Substring Without Repeating Characters

### 标准代码

```java
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> last = new HashMap<>();
    int left = 0;
    int ans = 0;

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (last.containsKey(c)) {
            left = Math.max(left, last.get(c) + 1);
        }
        last.put(c, right);
        ans = Math.max(ans, right - left + 1);
    }
    return ans;
}
```

### 一句话记忆

窗口不合法就移动左边界，Map 负责记状态。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Minimum Window Substring | 维护 need 和 valid |
| Permutation in String | 固定长度窗口比较计数 |
| Find All Anagrams | 固定长度窗口收集起点 |

### 常见坑点

- 更新 `left` 要取 `max`，不能往回退。
- 窗口题要分清固定窗口和可变窗口。

## 高频坑点

| 坑点 | 正确反应 |
|---|---|
| Two Sum 先 put | 可能使用同一元素 |
| 前缀和漏初始化 0 | 漏掉从 0 开始的子数组 |
| 有负数还用滑动窗口求和 | 改用前缀和 + HashMap |
| 字符集不止小写字母 | 用 HashMap |
| Map 计数为 0 还保留 | 比较状态时容易出错 |

## 面试速记

- 查出现：Set。
- 查次数：Map。
- 配对：查 complement。
- 子数组和为 K：前缀和 + Map。
- 子串频次：滑动窗口 + Map。

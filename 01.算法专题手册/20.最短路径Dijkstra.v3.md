# 最短路径 Dijkstra

## 本章目标

训练最短路选型：看到带权图最短路径，快速想到 Dijkstra；看到无权图，回到 BFS。

## 秒判卡

| 看到什么 | 直接想到 |
|---|---|
| 无权最短步数 | BFS |
| 边权都是 1 | BFS |
| 非负权最短路径 | Dijkstra |
| 0/1 边权 | 0-1 BFS |
| 有负权边 | Bellman-Ford / SPFA，不用 Dijkstra |
| 求从一个源点到所有点最短路 | Dijkstra |
| 网格代价最小路径 | Dijkstra / DP 看移动规则 |
| DAG 最短路 | 拓扑排序 DP |
| 最少操作次数 | BFS |
| 状态影响后续路径 | 状态图最短路 |
| 最多 K 次 / 钥匙 / 油量 / 权限 | dist 加维度 |
| 所有点对最短路 | Floyd |

---

## BFS vs Dijkstra

### BFS 适用场景

```text
边权相同。
通常每条边代价都是 1。
```

因此：

```text
层数 = 距离
```

所以：

- 先到的一定最短
- 用普通队列即可

典型题：

- 二叉树最短路径
- 网格最短步数
- 单词接龙
- 无权图最短路

---

### Dijkstra 适用场景

```text
边权非负，但不同。
```

此时：

```text
层数 ≠ 距离
```

不能按 BFS 分层推进。

必须：

- 每次取当前全局最小距离状态
- 用优先队列维护

典型题：

- Network Delay Time
- Cheapest Flights
- Path With Minimum Effort

---

### 本质区别

| 算法 | 核心 |
|---|---|
| BFS | 按层扩展 |
| Dijkstra | 按当前最短距离扩展 |

---

### 一句话判断

```text
无权最短路 → BFS

非负带权最短路 → Dijkstra
```

## 高频模板 1：堆优化 Dijkstra

### 秒判关键词

带权图、非负权、最短路径、网络延迟。

### 母题

LC743 Network Delay Time

### 标准代码

```java
public int networkDelayTime(int[][] times, int n, int k) {
    List<int[]>[] graph = new ArrayList[n + 1];
    for (int i = 1; i <= n; i++) graph[i] = new ArrayList<>();
    for (int[] t : times) {
        graph[t[0]].add(new int[]{t[1], t[2]});
    }

    int[] dist = new int[n + 1];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[k] = 0;

    PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[1]));
    pq.offer(new int[]{k, 0});

    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int u = cur[0], d = cur[1];
        if (d > dist[u]) continue;

        for (int[] edge : graph[u]) {
            int v = edge[0], w = edge[1];
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.offer(new int[]{v, dist[v]});
            }
        }
    }

    int ans = 0;
    for (int i = 1; i <= n; i++) {
        if (dist[i] == Integer.MAX_VALUE) return -1;
        ans = Math.max(ans, dist[i]);
    }
    return ans;
}
```

### 一句话记忆

每次从堆里取当前距离最小的点，松弛它的邻居。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Path With Minimum Effort | 距离定义成路径最大边权 |
| Cheapest Flights Within K Stops | 状态带剩余中转数 |
| Swim in Rising Water | 路径代价是经过格子的最大高度 |

### 常见坑点

- Dijkstra 要求边权非负。
- 堆里可能有旧距离，弹出时用 `if (d > dist[u]) continue`。

## 高频模板 2：网格 Dijkstra

### 秒判关键词

网格、每格代价、最小努力值、非均匀代价。

### 母题

LC1631 Path With Minimum Effort

### 标准思路代码

```java
public int minimumEffortPath(int[][] heights) {
    int m = heights.length, n = heights[0].length;
    int[][] dist = new int[m][n];
    for (int[] row : dist) Arrays.fill(row, Integer.MAX_VALUE);
    dist[0][0] = 0;

    PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[2]));
    pq.offer(new int[]{0, 0, 0});
    int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};

    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int r = cur[0], c = cur[1], effort = cur[2];
        if (r == m - 1 && c == n - 1) return effort;
        if (effort > dist[r][c]) continue;

        for (int[] d : dirs) {
            int nr = r + d[0], nc = c + d[1];
            if (nr < 0 || nr >= m || nc < 0 || nc >= n) continue;
            int next = Math.max(effort, Math.abs(heights[r][c] - heights[nr][nc]));
            if (next < dist[nr][nc]) {
                dist[nr][nc] = next;
                pq.offer(new int[]{nr, nc, next});
            }
        }
    }
    return 0;
}
```

### 一句话记忆

网格 Dijkstra 只是把坐标当节点。

### 高频变体

- Swim in Rising Water：距离是路径最大高度。
- Minimum Cost to Make Valid Path：边权 0/1，可用 0-1 BFS。

### 常见坑点

- 网格不是一定 BFS，代价不一样就要最短路。
- 到终点可以直接返回，因为堆弹出的就是当前最短。

## 高频模板 3：带状态最短路

### 秒判关键词

最多 K 次、剩余资源、状态影响路径。

### 母题

LC787 Cheapest Flights Within K Stops

---

### 核心思想

普通最短路：

```text
dist[node]
```

但如果：

```text
到达同一个节点时，
不同状态会影响后续决策
```

那么：

```text
dist[node]
不够
```

必须升级成：

```text
dist[node][state]
```

---

### 状态图思想

很多题里：

```text
真正的“节点”
并不是图节点本身
而是“状态”
```

例如：

```text
(node, usedEdges)
(node, remainingFuel)
(node, keysMask)
(node, obstaclesRemoved)
(node, color)
```

本质上：

```text
是在“状态图”上跑最短路
```

---

### LC787 的状态定义

```text
dist[node][usedEdges]
```

表示：

```text
从 src 出发，
使用 usedEdges 条边，
到达 node 的最小费用
```

---

### pq 状态定义

普通 Dijkstra：

```text
(node, cost)
```

787：

```text
(node, cost, usedEdges)
```

因为：

```text
状态已经不只是 node
```

---

### 初始化

```text
dist[src][0] = 0
```

表示：

```text
从 src 出发，
用了 0 条边，
费用是 0
```

---

### 为什么不能只用 dist[node]

因为：

```text
同一个 node，
不同 usedEdges，
后续还能走的路径不同。
```

错误剪枝会丢掉正确答案。

---

### 一句话记忆

如果：

```text
状态会影响未来决策
```

那么：

```text
状态必须进 dist
```

---

### 高频变体

- Shortest Path in a Grid with Obstacles Elimination
- Cheapest Flights
- 钥匙门问题
- 有油量限制的最短路
- 有免费次数的最短路

---

### 常见坑点

- 只用 `dist[node]` 会错误剪枝。
- K stops 和 K edges 差 1。
- dist 维度经常 off-by-one。
- pq 里的状态也必须扩维。

## 高频模板 4：0-1 BFS

### 秒判关键词

边权只有 0 和 1、最少代价。

### 母题

LC1368 Minimum Cost to Make at Least One Valid Path

### 标准思路

```text
用 Deque：
- 权重 0 的边放队首
- 权重 1 的边放队尾

效果类似 Dijkstra，但更快。
```

### 一句话记忆

0 代价插队头，1 代价排队尾。

### 高频变体

- 网格方向修改最小代价。
- 图中免费边和付费边混合。

### 常见坑点

- 权重不是 0/1 时不能用 0-1 BFS。
- 仍然要维护 dist。


---

### BFS / Dijkstra 的区别

如果：

```text
每次转移代价相同
```

→ BFS

如果：

```text
转移代价不同但非负
```

→ Dijkstra

## 高频坑点

| 坑点 | 正确反应 |
|---|---|
| 无权图用 Dijkstra | BFS 更简单 |
| 负权边用 Dijkstra | 不适用 |
| 堆里旧状态没跳过 | `d > dist[u]` continue |
| 状态限制没进 dist | dist 加维度 |
| K stops off-by-one | 明确中转数和边数 |

## 面试速记

- 无权最短路：BFS。
- 非负带权：Dijkstra。
- 0/1 权重：0-1 BFS。
- 负权：不用 Dijkstra。
- 状态会影响未来：把状态放进 dist。

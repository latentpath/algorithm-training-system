# 并查集 Union Find

## 本章目标

训练“动态连通性”的第一反应：看到合并集合、判断是否连通、朋友圈、冗余边，快速想到并查集。

## 秒判卡

| 看到什么 | 直接想到 |
|---|---|
| 合并两个集合 | Union Find |
| 判断两点是否连通 | `find(a) == find(b)` |
| 无向图连通分量 | 并查集 / DFS |
| 朋友圈 / 省份数量 | 并查集 |
| 冗余连接 / 成环 | union 前先查是否已连通 |
| 岛屿动态增加 | 并查集 |
| 有向图依赖顺序 | 拓扑排序，不是并查集 |

## 高频模板 1：标准并查集

### 秒判关键词

合并、查询连通、集合数量。

### 母题

LC547 Number of Provinces

### 标准代码

```java
class UnionFind {
    int[] parent;
    int[] size;
    int count;

    UnionFind(int n) {
        parent = new int[n];
        size = new int[n];
        count = n;
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            size[i] = 1;
        }
    }

    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }

    boolean union(int a, int b) {
        int ra = find(a), rb = find(b);
        if (ra == rb) return false;
        if (size[ra] < size[rb]) {
            parent[ra] = rb;
            size[rb] += size[ra];
        } else {
            parent[rb] = ra;
            size[ra] += size[rb];
        }
        count--;
        return true;
    }
}
```

### 一句话记忆

`find` 找老大，`union` 合并两个老大。

### 高频变体

| 题目 | 变化点 |
|---|---|
| Number of Provinces | 每条连接 union |
| Number of Connected Components | 对边做 union |
| Accounts Merge | 邮箱映射到账号并合并 |

### 常见坑点

- 合并的是根节点，不是原节点。
- `union` 成功时集合数才减一。

## 高频模板 2：冗余边 / 成环

### 秒判关键词

无向图加边、找导致环的边、冗余连接。

### 母题

LC684 Redundant Connection

### 标准代码

```java
public int[] findRedundantConnection(int[][] edges) {
    UnionFind uf = new UnionFind(edges.length + 1);
    for (int[] e : edges) {
        if (!uf.union(e[0], e[1])) {
            return e;
        }
    }
    return new int[0];
}
```

### 一句话记忆

加边前已经连通，这条边就是环边。

### 高频变体

- Graph Valid Tree：边数为 `n - 1` 且全部连通。
- Satisfiability of Equality Equations：先合并等式，再检查不等式。

### 常见坑点

- 并查集适合无向连通性。
- 有向图成环通常用拓扑或 DFS 三色。

## 高频模板 3：网格并查集

### 秒判关键词

二维网格连通、岛屿合并、坐标转编号。

### 母题

LC200 Number of Islands 的并查集版本

### 标准代码片段

```java
int id(int r, int c, int n) {
    return r * n + c;
}

// 遍历每个陆地格子，只和右、下方向的陆地 union
for (int r = 0; r < m; r++) {
    for (int c = 0; c < n; c++) {
        if (grid[r][c] != '1') continue;
        if (r + 1 < m && grid[r + 1][c] == '1') {
            uf.union(id(r, c, n), id(r + 1, c, n));
        }
        if (c + 1 < n && grid[r][c + 1] == '1') {
            uf.union(id(r, c, n), id(r, c + 1, n));
        }
    }
}
```

### 一句话记忆

二维坐标先压成一维编号，再合并相邻陆地。

### 高频变体

- Number of Islands II：动态加陆地，每次和四周 union。
- Surrounded Regions：边界 O 连到虚拟节点。

### 常见坑点

- 水格不应该计入岛屿数量。
- 动态岛屿中重复加同一格要跳过。

## 高频模板 4：带权并查集入口

### 秒判关键词

比例关系、等式关系、除法求值。

### 母题

LC399 Evaluate Division

### 标准思路

```text
parent[x] 表示 x 的父节点
weight[x] 表示 x / parent[x]
find(x) 时顺便压缩路径并更新 x / root
```

### 一句话记忆

普通并查集管连通，带权并查集顺便维护到根的比例。

### 高频变体

- Evaluate Division：字符串节点 + 比例权重。
- 等式约束：先合并，再查询关系。

### 常见坑点

- 带权并查集不适合初学阶段硬背完整代码，先掌握普通并查集。
- 权重含义必须固定，否则更新会乱。

## 高频坑点

| 坑点 | 正确反应 |
|---|---|
| `find` 不压缩路径 | 加路径压缩 |
| union 原节点 | union 根节点 |
| 集合数重复减少 | 只有根不同才减 |
| 有向图用并查集判依赖 | 改拓扑排序 |
| 网格编号错 | `r * n + c` |

## 一题多变体

```text
Number of Provinces
↓
Connected Components
↓
Accounts Merge
```

```text
Redundant Connection
↓

```

```text

```

## 面试速记

- 并查集：合并集合 + 查连通。
- `find`：找根，顺手路径压缩。
- `union`：根不同才合并。
- 无向图成环：union 前已连通。
- 二维网格：坐标转一维编号。

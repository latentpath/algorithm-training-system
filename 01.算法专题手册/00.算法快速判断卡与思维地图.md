# 算法快速判断卡与思维地图

> **核心目标**：10~30 秒内完成算法模型判断 + 建立完整结构化思维体系

---

## 第一部分：30秒面试判断卡（快速决策版）

**目标**：看到题目 10 秒内完成算法选型，不纠结、不试错

### ① 第一眼判断（问题本质）

**❓ 是否存在 / 频率 / 匹配 / 两数？**  
**→ HashSet / HashMap**  
- 出现过、统计次数、两数之和、子数组和为K  
**排除**：要求连续区间、最短路径、最值过程

**❓ 连续子数组 / 子串 / 区间？**  
**→ 双指针 / 滑动窗口 / 前缀和**  
- 连续 + 正数 → 滑动窗口  
- 连续 + 可能负数 → 前缀和 + Hash  
- 固定长度 → 固定窗口  
- 最长/最短满足条件 → 可变窗口  
**排除**：非连续

**❓ 最值 / 最优 / TopK？**  
**→ 贪心 / DP / 堆**  
- 局部最优可推广全局 → 贪心  
- 当前选择影响后续状态 → DP  
- 前K大/小、数据流、中位数 → 堆

**❓ 答案是数值阈值（最小化最大值 / 最大化最小值）？**  
**→ 二分答案**  
关键词：minimize the maximum、最小上限、最早时间、最小代价  
内部需配合可行性判定（BFS/DP/贪心等）

**❓ 连通性 / 遍历 / 最短路径？**  
**→ DFS / BFS / 并查集**  
- 无向连通/合并 → 并查集  
- 遍历/计数/染色 → DFS  
- 最短/层级/无权 → BFS  
- 有权非负 → Dijkstra / 0-1 BFS  
**强排除**：要“最早/最短/层级” → 直接否决 DFS

**❓ 顺序依赖 / 先修关系？**  
**→ 拓扑排序**（有向无环图）

---

### ② 模型级快速区分

| 对比             | 选择依据                          |
|------------------|-----------------------------------|
| 双指针 vs 滑动窗口 | 对撞（回文/有序）→ 双指针；连续可变 → 滑动窗口 |
| DFS vs BFS       | 所有方案/遍历 → DFS；最短/层级 → BFS |
| DP vs 贪心       | 当前最优依赖历史最优 → DP；不依赖 → 贪心 |
| 单调栈 vs 单调队列 | 下一个更大/更小 → 单调栈；窗口最值 → 单调队列 |

---

### ③ 三个必背秒否决信号

- **有负数** → 否决普通滑动窗口
- **非连续** → 否决窗口/双指针
- **有向 + 顺序依赖** → 否决并查集

---

### ④ 面试一句话总结

> “我会先判断题目核心结构（连续/连通/状态/最值/阈值等），再选择对应算法模型，最后落地实现。”

## 第二部分：算法思维地图（全景认知版）

### 一、算法的第一性问题

所有算法题本质只回答以下 **6 类核心问题** 之一：

| 核心问题       | 通俗理解           | 首选模型                     |
|----------------|--------------------|------------------------------|
| 是否存在 / 重复 | 有没有 / 出现几次   | HashSet / HashMap           |
| 连续区间       | 子数组 / 子串      | 双指针 / 滑动窗口 / 前缀和   |
| 最值 / 最优解   | 最大最小 / 最优方案 | 贪心 / DP / 堆               |
| 连通性 / 可达性 | 能不能走到 / 同一组 | DFS / BFS / 并查集           |
| 顺序依赖       | 谁必须在谁之前     | 拓扑排序                     |
| 距离 / 成本 / 时间 | 最短 / 最小代价   | BFS / Dijkstra / 0-1 BFS     |

### 三、核心算法思维分区

- **双指针 / 滑动窗口**：连续 + 可扩张 → 右扩左缩
- **前缀和 / 差分**：把区间操作从 O(n²) 降到 O(n)
- **DFS vs BFS**：所有方案 → DFS；最短/层级 → BFS
- **动态规划**：当前最优是否依赖之前的最优？
- **贪心**：局部最优能推出全局最优且无反例
- **单调结构**：下一个更大/更小（栈）、窗口极值（队列）
- **并查集**：动态连通性，不关心路径
- **拓扑排序**：有向无环图的顺序依赖
